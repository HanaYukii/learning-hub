# 數位 DP

**要解什麼**:「$[L,R]$ 內滿足某個**數位條件**(各位和、含哪些數字、相鄰位關係、可被某數整除…)的數有幾個」,而且 $R$ 大到不能一個個枚舉(常直接是字串)。

**通用手法**:只要會算 $\mathrm{solve}(N)=[0,N]$ 內的個數,答案就是 $\mathrm{solve}(R)-\mathrm{solve}(L-1)$。而 $\mathrm{solve}(N)$ 用「**從高位往低位、一位一位填**」的記憶化 DFS 來數。

## 入門例:數字和為 S 的個數

> 求 $[0,N]$ 中「各位數字和恰為 $S$」的整數個數($N$ 可以很大,字串讀入)。

一位一位往下填,只要記住三件事:填到**第幾位** `pos`、目前**數字和** `sum`、是不是**還貼著 $N$ 的上界** `tight`(貼著的話這位不能超過 $N$ 的對應位)。填完(`pos` 到底)看 `sum == S` 就好:

```cpp
int S;                              // 目標數字和
vector<int> d;                      // d[0] = 最高位
ll memo[20][200]; bool vis[20][200];   // 狀態:pos × sum

ll dfs(int pos, int sum, bool tight) {
    if (sum > S) return 0;                        // 剪枝:已超過
    if (pos == (int)d.size()) return sum == S;    // 填完 → 看是否達標
    if (!tight && vis[pos][sum]) return memo[pos][sum];   // 只快取「自由狀態」
    int hi = tight ? d[pos] : 9;                  // 貼緊時這位上限是 d[pos],否則 9
    ll res = 0;
    for (int nd = 0; nd <= hi; nd++)
        res += dfs(pos + 1, sum + nd, tight && nd == hi);
    if (!tight) { vis[pos][sum] = true; memo[pos][sum] = res; }
    return res;
}
// solve(N):把 N 的每位放進 d[](d[0]=最高位),回傳 dfs(0, 0, true)
// 答案 [L,R] = solve(R) − solve(L−1)
```

這就是數位 DP 的全部骨架。難的不是這段 code,而是下面這個問題:**`tight` 為什麼不放進 `memo`?**

## 核心:哪些維度該進 memo key

把狀態分成兩類:

- **內容維度**(`pos`、`sum`)——真正定義子問題「剩下怎麼填」的東西,**要進 memo key**。
- **唯一路徑維度**(`tight`)——貼著上界與否,**不進 key**,只當快取的開關。

為什麼 `tight` 不進 key?兩個理由:

1. **它只會被走到一次**。`tight` 為真代表「前面每一位都剛好等於 $N$ 的那一位」——這種前綴只有一條(就是 $N$ 自己)。存進表也永遠不會有第二次來讀。
2. **混用還會算錯**。自由狀態(`tight=false`)算的是「剩下每位 $0\sim9$ 隨便填」的方法數;貼緊狀態算的是「剩下被 $N$ 卡住」的方法數。**同一個 `(pos, sum)`,這兩個答案不一樣**:例如 $N=531$,走到最後一位——自由能填 $0\sim9$(10 種),貼緊只能填 $0\sim1$($\le$ 個位)。若共用同一格,自由先寫 10、貼緊來讀就錯拿 10。

所以慣例是:**只有自由狀態(`!tight`)才讀 / 寫快取**。而自由狀態的答案只跟 `(pos, sum)` 有關、跟前面怎麼填到這裡無關 → 大量前綴撞在一起,快取才真正省時間。

> ⚠️ 讀和寫的 guard 要**對稱**:`if(!tight)` 讀、也用同一條件寫。只擋一邊會出錯。

## 前導零:什麼時候需要 `lead`

上面數字和的例子不用管前導零(補在前面的 0 對「數字和」沒影響)。但**當狀態跟「用了哪些數字 / 某數字出現幾次」有關時**,前導零就會搗亂——你不希望 `6` 被當成 `06` 而多算一個數字 `0`。

這時加一個 `lead`(是否還在前導零)維度,規則:**還在前導零時,不更新內容維度**。

```cpp
bool nlead = lead && (nd == 0);                 // 這位填 0 且之前全是前導零
int nmask  = nlead ? mask : (mask | (1 << nd)); // 前導零不算「用過這個數字」
```

`lead` 跟 `tight` 一樣是「唯一路徑、只走一次」的維度,同樣**不進 key、只 guard**(`if(!tight && !lead)`)。

**小技巧**:若某個內容維度剛好在「還沒放真數字」時等於定值,就能用它代替 `lead`。例如用 `mask`(用過的數字集合)時,`mask == 0 ⟺` 前導零(mask 只增不減,放任何真數字就 $\ne 0$),可省掉 `lead` 這一維。反例:只追蹤 `mod`,`mod == 0` 分不出「前導零」還是「真的整除」,就必須留 `lead`。

## 多查詢時的加碼

要算很多筆不同 $N$(或很多組 $[L,R]$)時,注意:**自由狀態的答案跟 $N$ 無關**——「剩幾位自由填、目前狀態如何」的方法數,不管 $N$ 多長都一樣。所以若你用「**剩幾位**」而不是絕對 `pos` 當索引,這份快取可以**跨所有查詢共用、永不清表**。（把 `tight` 放進 key 的寫法,每換一個 $N$ 都得 `memset` 重來——這就是把 tight 留在 key 外最大的好處,不只是省記憶體。）

## 例題 / 變形

- **ABC465 E — Digit Circus**（`AtC~1400`）:求 $[1,N]$($N<10^{500}$)中「恰滿足一條件」的個數:(1) 3 的倍數、(2) 含數字 3、(3) 恰用 3 種相異數字。狀態 `mask`(用過數字)× `mod`(數字和 mod 3)× `tight`,用 `mask==0` 代 `lead`,末端判「三條件恰一成立」。作法見 [ABC465 digest](/cp/contests/2026-07-04-ac-abc465)。
- **HDU 2089 不要 62**:數「不含 4、也不出現子串 62」的個數。內容維度 = `prev`(前一位是不是 6);此題 0 可隨便放、不影響條件,所以**不需要 `lead`**——需不需要 `lead` 由題目決定。
- **CF 1245F Daniel and Spring Cleaning**:對兩個上界同時跑,數 $a+b=a\oplus b$ 的對數。
- 換 `state` 與末端判定,骨架不變:各位數字和、某數字出現次數、可被某數整除、相鄰位單調…都是這一招。
