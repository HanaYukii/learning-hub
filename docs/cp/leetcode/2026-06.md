### [Q4. Maximum Score with Co-Prime Element](https://leetcode.com/problems/maximum-score-with-co-prime-element/)
`數論` `容斥` `LC~2390`

::: info 題意
給定長度 n 的陣列 nums 與整數 maxVal。可將任意元素改成任一 ≤maxVal 的正整數,每次改動花費 1。改完後須選一個 index i,使 nums[i] 與其餘所有 nums[j] 互質(gcd=1)。設 selectedValue 為 nums[i] 最終值、modificationCost 為被改動元素總數,score = selectedValue − modificationCost。求最大 score。約束 1<=n<=1e5,1<=nums[i]<=1e5,1<=maxVal<=1e5。

例:nums=[3,4,6], maxVal=5。取 v=5:無元素含 5 的因子(conflictAll=0)→改一元素成 5,cost=1,score=4;或保留原生 6:conflictAll(6)=3(3,4,6 皆與 6 有公因數),score=6−(3−1)=4。答案 4。
:::

::: tip 作法
固定「被選元素的最終值」$v$:所有與 $v$ **不互質**的元素都得改掉(改成 $1$ 最省)。令 **$C(v)=$ 陣列中與 $v$ 不互質的元素數**(code 裡的 `conflict`),整題化成「枚舉候選 $v$、算最小改動成本」。

**核心:$C(v)$ 為何要容斥**——$\gcd(x,v)>1 \iff x$ 被 $v$ 的**某個質因子**整除。設 $v$ 的相異質因子 $p_1,\dots,p_k$,令 $A_i=\{x : p_i \mid x\}$,要的是聯集 $C(v)=\lvert A_1\cup\cdots\cup A_k\rvert$。直接相加會把「同時被多個 $p_i$ 整除」者重複計 → **容斥**:

$$C(v)=\sum_{\varnothing\ne S\subseteq\{1,\dots,k\}}(-1)^{|S|+1}\Bigl\lvert\bigcap_{i\in S}A_i\Bigr\rvert .$$

質數兩兩互質 ⇒「同時被 $S$ 內每個 $p_i$ 整除」$\iff$「被乘積整除」,故 $\bigl\lvert\bigcap_{i\in S}A_i\bigr\rvert=\mathrm{cnt}\!\left[\prod_{i\in S}p_i\right]$,其中 **$\mathrm{cnt}[d]=$ 被 $d$ 整除的元素數**,用調和篩 $\mathrm{cnt}[d]=\sum_{d\mid m}\mathrm{freq}[m]$ 一次算好($O(V\log V)$)。**奇數大小子集加、偶數減**(即 code 的 `bits & 1 ? +cnt : -cnt`)。$v\le10^5$ 相異質因子 $\le 6$ 個($2\cdot3\cdot5\cdot7\cdot11\cdot13=30030$,七個就 $>10^5$)→ 每個 $v$ 至多 $2^6-1=63$ 項。

**例** $C(6)$:$6=2\cdot3$ → $\mathrm{cnt}[2]+\mathrm{cnt}[3]-\mathrm{cnt}[6]$;於 $\{3,4,6\}$ 得 $2+2-1=3$($3,4,6$ 全與 $6$ 不互質)。

**選定 $v$ 後最小成本分三型**:(1) 有原生元素 $=v$、拿它當被選不改它 → $\mathrm{cost}=C(v)-[\,v>1\,]$(它自己也被 $C$ 算到、扣回;此型 $v$ 可超過 $maxVal$,不改就不受上限);(2) 無原生 $v$ 但有衝突元素 → 把其中一顆改成 $v$ 當被選 → $\mathrm{cost}=C(v)$;(3) 全無衝突 → 改一顆互質元素成 $v$ → $\mathrm{cost}=1$。

故答案 $=$ 掃全部取 $\max$:$v\in[1,maxVal]$ 用 $v-\max(C(v),1)$(涵蓋 (2)(3)),原生值 $w$ 用 $w-(C(w)-[\,w>1\,])$(涵蓋 (1))。總 $O(V\log V + V\cdot 2^6)$。

> 要對**所有 $v$** 一次算好 $C(v)$,可把逐 $v$ 的質因子容斥用 **μ 除數篩**批次做掉($\mathrm{coprime}[v]=\sum_{d\mid v}\mu(d)\,\mathrm{cntDiv}[d]$,$O(V\log V)$)——見技巧卡 [莫比烏斯反演 / μ 除數篩](/cp/techniques/mobius-inversion)。

```cpp
int V = max(maxVal, *max_element(nums.begin(), nums.end()));
vector<int> freq(V + 1, 0), cnt(V + 1, 0), spf(V + 1);
for (int x : nums) freq[x]++;
for (int d = 1; d <= V; d++)
    for (int m = d; m <= V; m += d) cnt[d] += freq[m];  // cnt[d]=#divisible by d
iota(spf.begin(), spf.end(), 0);
for (int i = 2; (long long)i * i <= V; i++)
    if (spf[i] == i)
        for (int j = i * i; j <= V; j += i) spf[j] = min(spf[j], i);
auto conflict = [&](int v) {  // # sharing a factor with v
    vector<int> ps;
    for (int t = v; t > 1;) {
        int p = spf[t];
        ps.push_back(p);
        while (t % p == 0) t /= p;
    }
    int k = ps.size(), tot = 0;
    for (int mask = 1; mask < (1 << k); mask++) {
        long long prod = 1;
        int bits = 0;
        for (int b = 0; b < k; b++)
            if (mask >> b & 1) {
                prod *= ps[b];
                bits++;
            }
        tot += (bits & 1 ? cnt[prod] : -cnt[prod]);
    }
    return tot;
};
long long ans = LLONG_MIN;
for (int v = 1; v <= V; v++) {
    if (v > maxVal && !freq[v]) continue;
    int c = conflict(v);
    if (v <= maxVal) ans = max(ans, (long long)v - max(c, 1));  // create v
    if (freq[v]) ans = max(ans, (long long)v - (c - (v > 1 ? 1 : 0)));
}  // keep native v
```
:::

### [Q4. Maximum Sum of M Non-Overlapping Subarrays II](https://leetcode.com/problems/maximum-sum-of-m-non-overlapping-subarrays-ii/)
`dp` `二分` `資料結構` `LC~2611`

::: info 題意
與 Q3 完全同題(選至少1、至多m個互不重疊、長度∈[l,r]的子陣列使總和最大,回傳最大總和;全負時仍須選一段),但約束放大:**n≤1e5**,|nums[i]|≤1e5,1≤m≤n,1≤l≤r≤n。此時 Q3 的 O(nm) 在 m=Θ(n) 時達 1e10 會 TLE,需要更優做法。

例:nums=[-3,-4,-1], m=2, l=1, r=2。全部子陣列和 ≤0,λ=0 的自由最優取 0 段 → 觸發 forced 分支,答案取單一最佳子陣列 [-1]=**-1**。這條剛好卡「至少一段 + 全負」邊界,是我第一版 WQS(誤用 g+λ·c、且在平台/平手時錯判 0 段)對拍掛掉之處,修正後對過。
:::

::: tip 作法
設 **f(k)=恰好 k 段的最大總和**,可證 f 對 k **凹**(邊際遞減)。用 **WQS / Aliens(Lagrangian)二分**:每段扣罰金 λ 做「無段數限制」的 O(n) 滑窗 DP,並同時追蹤達成最優值的**最小段數**;整數二分最小的 λ 使最優段數 ≤ m,答案 = g(λ)+λ·m(即 f(min(m,k*)))。「至少一段」與全負邊界靠 **λ=0 的自由最優**判斷:自由最優取 0 段 ⇒ 所有子陣列和 ≤0 ⇒ 答案為單一最佳子陣列;若自由最優段數 k*≤m 則直接回 f(k*)。總 **O(n log V)**,V≈n·max|nums|≈2e10。

```cpp
// P[i]=前綴和(P[0]=0)。solve(lam) 回 {>=0段、每段扣 lam 的最大值, 達成該值的最小段數}
array<ll, 2> solve(ll lam) {
    vector<ll> f(n + 1, 0);
    vector<int> c(n + 1, 0);
    deque<int> dq;
    auto key = [&](int j) { return f[j] - P[j]; };  // dp[i]=max(dp[i-1], P[i]-lam+max_{j∈[i-r,i-l]}key(j))
    for (int i = 1; i <= n; i++) {
        int j = i - l;
        if (j >= 0) {
            while (!dq.empty() && (key(dq.back()) < key(j) || (key(dq.back()) == key(j) && c[dq.back()] >= c[j]))) dq.pop_back();
            dq.push_back(j);
        }
        while (!dq.empty() && dq.front() < i - r) dq.pop_front();
        f[i] = f[i - 1];
        c[i] = c[i - 1];  // 跳過 i
        if (!dq.empty()) {
            int t = dq.front();
            ll v = P[i] - lam + key(t);
            int cc = c[t] + 1;
            if (v > f[i] || (v == f[i] && cc < c[i])) {
                f[i] = v;
                c[i] = cc;
            }
        }  // 平手取最小段數
    }
    return {f[n], c[n]};
}
// driver:
auto A = solve(0);                    // 自由最優
if (A[1] == 0) return bestSingleSeg;  // 全部和<=0 → 取一段最不負的
if (A[1] <= m) return A[0];           // k*<=m
ll lo = 1, hi = 2e10;                 // 找最小 lam 使段數<=m
while (lo < hi) {
    ll mid = (lo + hi) / 2;
    if (solve(mid)[1] <= m)
        hi = mid;
    else
        lo = mid + 1;
}
return solve(lo)[0] + lo * (ll)m;  // g(lam)+lam*m = f(m)
```
:::

### [Q4. Maximum Subarray Sum After at Most K Swaps](https://leetcode.com/problems/maximum-subarray-sum-after-at-most-k-swaps/)
`貪心` `資料結構` `二分` `LC~2672`

::: info 題意
給整數陣列 nums(n≤1500)與整數 k。可做至多 k 次交換,每次交換選任意兩個索引 i,j 並交換 nums[i]、nums[j] 的值。求交換完成後,某個連續且非空子陣列之和的最大可能值。約束:nums[i]∈[−1e5,1e5],0≤k≤n。

例:nums=[1,−1,0,2],k=1。窗 {1,−1}(base 0),窗外 {0,2};t=1 把窗內最小 −1 換成窗外最大 2 → 保留的 1 + 換入的 2 = 3,即答案 3。全陣列窗 base=2 無外可換,故最優仍是 3。
:::

::: tip 作法
固定子陣列窗 [l,r]:一次交換恰把一個窗外元素換進來、一個窗內元素換出去,做 t 次後窗內內容 = 「窗內最大的 len−t 個」+「窗外最大的 t 個」。要最大化就把窗內最小的 t 個換成窗外最大的 t 個;因 (第 t 大窗外 − 第 t 小窗內) 隨 t 單調遞減,累加到差 ≤0 或 t=k 為止即該窗最優(增益對 t 為凹)。答案 = 枚舉所有 O(n²) 窗取最大。加速:兩棵值域 Fenwick 各維護窗內/窗外的順序統計與前綴和,滑動 r 時把 nums[r] 由窗外移入窗內,對每窗二分最大可行 t → O(n² log² n),n≤1500 綽綽有餘。

```cpp
long long best = LLONG_MIN;
for (int l = 0; l < n; l++) {
    IN.clear();
    OUT = allValues;  // Fenwick over compressed vals
    long long base = 0;
    for (int r = l; r < n; r++) {
        OUT.erase(a[r]);
        IN.insert(a[r]);
        base += a[r];
        int lo = 0, hi = min({k, r - l + 1, n - (r - l + 1)});  // 二分最大有益 t
        while (lo < hi) {
            int t = (lo + hi + 1) / 2;
            if (OUT.kthLargest(t) > IN.kthSmallest(t))
                lo = t;
            else
                hi = t - 1;
        }
        long long gain = OUT.sumTop(lo) - IN.sumBottom(lo);
        best = max(best, base + gain);
    }
}
```
:::

### [Q3. Maximize Sum of Device Ratings](https://leetcode.com/problems/maximize-sum-of-device-ratings/)
`貪心` `LC~1880`

::: info 題意
給 m×n 矩陣 units,units[i][j] 是第 i 個裝置第 j 個單元的容量;每個裝置恰有 n 個單元,評分 = 該裝置所有單元容量的最小值。操作可做任意次:選一個「先前未當過來源」的裝置 i,從 i 移除恰好一個單元、加到另一個不同裝置,並把 i 標記為已用(此後不得再當來源);接收不受次數限制,空裝置評分為 0。求所有操作後評分總和的最大值。約束:m,n≥1,m·n≤2e5,容量∈[1,1e5]。

例:units=[[1,3],[2,2]]:a=[1,2],R(次小)=[3,2];ΣR=5,minR=2,amin=1 → 5−2+1=4(把裝置0的1丟給裝置1:[3]與[2,2,1]→3+1=4)。
:::

::: tip 作法
接收單元永不提高評分(min 只會變小),故每個裝置最多當一次來源、丟掉自己的最小值,評分由最小值 a_i 升到次小值 R_i(n=1 時 R_i=0)。所有被丟出的值都 ≥ 全域最小 amin,因此可全部倒進「R 最小的那台」當唯一代罪羊,總損失恰好一次。**答案 = ΣR_i − minR_i + amin**;n=1 時無操作有利,直接回原值總和。亮點:把所有被迫的傾倒集中到單一最小評分裝置,代價收斂成單一項 (minR − amin)。

```cpp
long long sumR = 0, minR = LLONG_MAX, amin = LLONG_MAX;
for (auto& d : units) {
    sort(d.begin(), d.end());
    if (n == 1) {
        ans += d[0];
        continue;
    }
    long long a = d[0], R = d[1];  // 最小、次小
    sumR += R;
    minR = min(minR, R);
    amin = min(amin, a);
}
// n>=2: ans = sumR - minR + amin
```
:::

### [Q4. Minimum Time to Reach Target With Limited Power](https://leetcode.com/problems/minimum-time-to-reach-target-with-limited-power/)
`最短路` `圖論` `狀態擴充` `分層圖` `LC~2103`

::: info 題意
有向加權圖 n 個點(1..1000);edges[i]=[u,v,t] 表 u→v 費 t 秒(0<=邊數<=1000,1<=t<=1e9)。初始 power(1..1000);cost[u]\(1..2000)為從 u 沿任一出邊轉發訊號所需 power。訊號 time 0 於 source、持 power。規則:剩餘 power>=cost[u] 才能走 u 的出邊;離開 u 時 power 減 cost[u];抵達節點不耗 power;走邊 [u,v,t] 使 time 加 t。輸出長度 2 陣列:answer[0]=抵達 target 的最短 time;answer[1]=所有達成該最短 time 的路徑中最大剩餘 power;不可達回 [-1,-1]。約束:1<=n<=1000。

例:Example1(n=5,power=4,cost=[2,3,1,1,1],0→4):0→2→3→4 耗 cost0+cost2+cost3=4、剩 0、time=3;0→1→4 不可(離開 0 後剩 2<cost1=3)→ [3,0]。Example2 source=target→ [0,3]。Example3 不連通→ [-1,-1]。三例與 30000 筆隨機案例對拍 DFS 暴力(power>=1 保證路徑有界)皆 0 mismatch。
:::

::: tip 作法

> **Q3** [Maximum Subarray Sum After Multiplier](https://leetcode.com/problems/maximum-subarray-sum-after-multiplier/) `LC~1981` — 核心是教科書級的三段式狀態機 Kadane(未轉換/轉換中/已轉換),對 IM 2347 屬顯然套路;唯一非顯然點是「恰好一次操作不會讓答案變差」的論證(乘正段、除負元素必有一利,故可直接對兩操作各跑 Kadane 取 max、無須補『操作置於窗外』項),但這是窄面向的分類分析,可遷移價值不足以收錄。(掃過,無新亮點)
