### [Q4. Count Distinct Ways to Form Target from Two Strings](https://leetcode.com/problems/count-distinct-ways-to-form-target-from-two-strings/)
`dp` `組合計數` `容斥` `LC~2128`

::: info 題意
給三個字串 word1、word2、target(長度均 ≤100,僅小寫)。對 target 的每個字元,從 word1 或 word2 中選一個相同字元;各自字串內選到的索引須嚴格遞增;且 word1、word2 至少各被選一次。兩方案相異的定義:存在某個 target 位置,其字元來自不同字串或不同索引。求方案數 mod 1e9+7。

例:word1="ab", word2="cde", target="ace"。只用 word1:"ace" 非 "ab" 的子序列 → 0;只用 word2:'a'∉"cde" → 0;可用兩串:唯一 a←w1[0], c←w2[0], e←w2[2] → 1。答案 = 1−0−0 = 1 ✓(對照 example 1「abc/bac/abc→5」亦與暴力一致)。
:::

::: tip 作法
**容斥:答案 =(可用兩串)−(只用 word1)−(只用 word2)**,把「至少各選一個」轉成無限制減掉退化情形。可用兩串以 dp[i][j] = 已配好 target 某前綴、下一個 word1 索引 ≥ i、word2 索引 ≥ j 的方案數;逐 target 字元用前綴和轉移(取自 word1:nd[p+1][j] += Σ_{i≤p} dp[i][j];取自 word2 對稱)。單串項 = target 作為該串子序列的嵌入計數(標準一維 DP)。時間 O(T·|w1|·|w2|),空間 O(|w1|·|w2|)。

```cpp
const long long MOD = 1e9 + 7;
int n1 = w1.size(), n2 = w2.size(), T = t.size();
vector<vector<long long>> dp(n1 + 1, vector<long long>(n2 + 1, 0));
dp[0][0] = 1;
for (char c : t) {
    vector<vector<long long>> nd(n1 + 1, vector<long long>(n2 + 1, 0));
    for (int j = 0; j <= n2; j++) {
        long long s = 0;  // 取自 word1:對 i 做前綴和
        for (int p = 0; p < n1; p++) {
            s = (s + dp[p][j]) % MOD;
            if (w1[p] == c) nd[p + 1][j] = (nd[p + 1][j] + s) % MOD;
        }
    }
    for (int i = 0; i <= n1; i++) {
        long long s = 0;  // 取自 word2:對 j 做前綴和
        for (int q = 0; q < n2; q++) {
            s = (s + dp[i][q]) % MOD;
            if (w2[q] == c) nd[i][q + 1] = (nd[i][q + 1] + s) % MOD;
        }
    }
    dp = move(nd);
}
long long tot = 0;
for (auto& r : dp)
    for (long long v : r) tot = (tot + v) % MOD;
auto sub = [&](string& w) { vector<long long> f(T+1,0); f[0]=1;   // target 為 w 子序列的嵌入數
    for(char ch:w) for(int k=T-1;k>=0;k--) if(t[k]==ch) f[k+1]=(f[k+1]+f[k])%MOD; return f[T]; };
return (int)(((tot - sub(w1) - sub(w2)) % MOD + MOD) % MOD);
```

### [Q4. Palindromic Subarray Sum](https://leetcode.com/problems/palindromic-subarray-sum/)
`字串` `貪心` `二分` `LC~2202`

::: info 題意
給正整數陣列 nums(值皆為正),回傳「是回文的連續子陣列」的最大元素和。回文指該子陣列作為數值序列正著讀與反著讀相同;長度為 1 的子陣列也算回文。約束:1<=n<=1e5,1<=nums[i]<=1e9(和可達 1e14,需 64 位)。

例:nums=[1,2,3,2,1,5,6]。以 index 2(值 3)為中心的最長奇回文半徑 2 → 子陣列 [1,2,3,2,1],和 = 1+2+3+2+1 = 9,為所有回文中最大。
:::

::: tip 作法
因元素全為正,對固定回文中心,半徑越大和越大 → 每個中心只需取**最長**回文即可。用 **Manacher**(把字元相等改成整數相等)在 O(n) 求每個奇/偶中心的最長回文半徑,配合前綴和算出該回文的和,取全域最大。替代解:雜湊 + 對每中心二分半徑,O(n log n)。

```cpp
long long solve(vector<int>& a) {
    int n = a.size();
    vector<long long> pre(n + 1, 0);
    for (int i = 0; i < n; ++i) pre[i + 1] = pre[i] + a[i];
    auto sum = [&](int l, int r) { return pre[r + 1] - pre[l]; };
    long long ans = LLONG_MIN;
    // odd centers
    vector<int> d1(n);
    for (int i = 0, l = 0, r = -1; i < n; ++i) {
        int k = (i > r) ? 1 : min(d1[l + r - i], r - i + 1);
        while (i - k >= 0 && i + k < n && a[i - k] == a[i + k]) ++k;
        d1[i] = k;
        if (i + k - 1 > r) {
            l = i - k + 1;
            r = i + k - 1;
        }
        ans = max(ans, sum(i - (k - 1), i + (k - 1)));
    }
    // even centers (pair between i-1,i)
    vector<int> d2(n);
    for (int i = 0, l = 0, r = -1; i < n; ++i) {
        int k = (i > r) ? 0 : min(d2[l + r - i + 1], r - i + 1);
        while (i - k - 1 >= 0 && i + k < n && a[i - k - 1] == a[i + k]) ++k;
        d2[i] = k;
        if (i + k - 1 > r) {
            l = i - k;
            r = i + k - 1;
        }
        if (k > 0) ans = max(ans, sum(i - k, i + k - 1));
    }
    return ans;
}
```

### [Q3. Divisible Game](https://leetcode.com/problems/divisible-game/)
`dp` `數論` `貪心` `LC~1944`

::: info 題意
給長度 n 的正整數陣列 nums。Alice 選整數 k>1 與區間 0<=l<=r<n。對區間內每個 i:若 k | nums[i] 則 Alice 加 nums[i],否則 Bob 加 nums[i]。分差 = Alice − Bob;Alice 要最大化分差,並在所有達到最大分差的 k 中取最小者。回傳 (最大分差 × 該 k) mod 1e9+7(分差可為負,用非負取模)。約束:1<=n<=1000,1<=nums[i]<=1e6。

例:nums=[1,4,6,8]。質數候選 {2,3}。k=2:a=[-1,4,6,8],最大子段=4+6+8=18;k=3:a=[-1,-4,6,-8]=6。最大分差 18 於 k=2 → 18×2=36。
:::

::: tip 作法
固定 k 後令 a_i = +nums[i]\(若 k|nums[i])否則 −nums[i],最大分差就是**非空最大子段和 Kadane**。**關鍵亮點:若 k'|k 則逐項 a_i^{(k')} >= a_i^{(k)},故 f(k') >= f(k) → 最優 k 必落在質數**;且最小的最大化 k 一定是質數(任何合數的最小質因子更小又同樣達最大)。只需列舉「整除某元素的質數」各跑一次 Kadane,取最大分差、同分取最小質數;全為 1 的特例回傳 −1×2。複雜度 O(√V·n + P·n),P 為相異質因子數(每數 <=7 個,實務極小)。

```cpp
long long solve(vector<int>& nums) {
    const long long MOD = 1e9 + 7;
    long long bestD = LLONG_MIN, bestK = -1;
    set<int> P;
    for (int v : nums) {
        int x = v;
        for (int d = 2; (long long)d * d <= x; ++d)
            if (x % d == 0) {
                P.insert(d);
                while (x % d == 0) x /= d;
            }
        if (x > 1) P.insert(x);
    }
    if (P.empty()) {
        bestD = -*min_element(nums.begin(), nums.end());
        bestK = 2;
    }
    for (int k : P) {  // 升序 → 只在嚴格更大時替換,保留最小 k
        long long cur = 0, best = LLONG_MIN;
        for (int v : nums) {
            long long a = (v % k == 0) ? v : -(long long)v;
            cur = max(a, cur + a);
            best = max(best, cur);
        }
        if (best > bestD) {
            bestD = best;
            bestK = k;
        }
    }
    return ((bestD % MOD + MOD) % MOD) * (bestK % MOD) % MOD;
}
```

### [Q4. Maximum Consistent Columns in a Grid](https://leetcode.com/problems/maximum-consistent-columns-in-a-grid/)
`dp` `LC~2014`

::: info 題意
給 m×n 整數網格 grid 與整數 limit(1≤m,n≤250,|grid[i][j]|≤1e5,0≤limit≤1e5)。可刪除任意欄(至少保留一欄),保留欄的相對順序不變。網格稱為 consistent:對每列 i、及保留後每對「相鄰」欄 a<b(即保留序列中連續的兩欄),皆滿足 |grid[i][b]-grid[i][a]|≤limit。求最多能保留幾欄。

例:grid=[[1,-1,1],[2,2,2]], limit=1。欄向量 c0=(1,2),c1=(-1,2),c2=(1,2)。相容性:(0,1) |−1−1|=2>1✗、(1,2) |1−(−1)|=2>1✗、(0,2) |1−1|=0 且 |2−2|=0 ✓。最長鏈 {0,2} 長 2 → 答案 2。
:::

::: tip 作法
關鍵觀察:『相鄰』只約束保留序列中**連續**的兩欄,故等價於在欄上建 DAG——邊 a→b(a<b)存在 ⟺ 兩欄相容(所有列皆 |grid[i][b]-grid[i][a]|≤limit)——求**最長鏈**。**dp[b]=1+max{dp[a]:a<b 且相容}**,答案取 max;相容判定掃全部列。總複雜度 O(n²·m)≈1.5e7,直接過。

```cpp
int maxColumns(vector<vector<int>>& g, int limit) {
    int m = g.size(), n = g[0].size();
    vector<int> dp(n, 1);
    int ans = 1;
    for (int b = 0; b < n; b++) {
        for (int a = 0; a < b; a++) {
            bool ok = true;
            for (int i = 0; i < m && ok; i++)
                if (abs(g[i][b] - g[i][a]) > limit) ok = false;
            if (ok) dp[b] = max(dp[b], dp[a] + 1);
        }
        ans = max(ans, dp[b]);
    }
    return ans;  // O(n^2*m)
}
```

### [Q4. Minimum Cost to Convert String III](https://leetcode.com/problems/minimum-cost-to-convert-string-iii/)
`dp` `字串` `LC~est2300`

::: info 題意
給等長小寫字串 source、target(長 n≤5000)。給 rules[i]=[pattern,replacement]\(等長,長≤20;pattern 至少一個字母、至多 5 個 '*')與 costs[i]。套用規則 i 於起點 l:區間 [l,l+len-1] 內所有位置皆「未曾被任何規則用過」,且每個 pattern[j] 為 '*' 或等於當前字元;把該段整段換成 replacement,花費 costs[i] + pattern 中 '*' 數;被用過的位置永久鎖定不可再被覆蓋。求把 source 變成 target 的最小總花費,不可能回 -1。

例:source="test",target="next",rule ["*e*t"→"next"],cost 4。pattern 兩個 '*' 分別吃掉 't'、's','e'/'t' 定位對上;replacement "next"==target;花費 4+2=6。
:::

::: tip 作法
**關鍵觀察:每個位置至多被覆蓋一次(用過即鎖)**,因此不像 String I/II 存在「同位置連鎖轉換→建圖跑最短路」的結構,而是把 [0,n) 切成若干**不相交區間**,未覆蓋的位置必須 source==target。每條規則花費固定 = costs+#'*',且套用時比對的「當前字元」永遠是原始 source(未鎖位置沒被改過)⇒ 套用順序無關。線性前綴 DP:dp[i]=完成前綴 [0,i) 的最小花費;轉移為「source[i]==target[i] 時可空過 dp[i+1]」或「套某規則(replacement 逐位==target 段 且 pattern 對 source 段相符)跳 dp[i+len]」。複雜度 O(n·R·L)≈5000·200·20≈4e7。

```cpp
int minCost(string s, string t, vector<vector<string>>& ru, vector<int>& c) {
    if (s.size() != t.size()) return -1;
    int n = s.size(), R = ru.size();
    const long long INF = 1e18;
    vector<long long> dp(n + 1, INF);
    dp[0] = 0;
    vector<long long> cost(R);
    for (int r = 0; r < R; r++) cost[r] = c[r] + count(ru[r][0].begin(), ru[r][0].end(), '*');
    for (int i = 0; i < n; i++) {
        if (dp[i] == INF) continue;
        if (s[i] == t[i]) dp[i + 1] = min(dp[i + 1], dp[i]);
        for (int r = 0; r < R; r++) {
            auto& p = ru[r][0];
            auto& rep = ru[r][1];
            int L = p.size();
            if (i + L > n) continue;
            bool ok = true;
            for (int j = 0; j < L && ok; j++)
                if (rep[j] != t[i + j]) ok = false;  // replacement==target 段
            for (int j = 0; j < L && ok; j++)
                if (p[j] != '*' && p[j] != s[i + j]) ok = false;  // pattern 對 source 段
            if (ok) dp[i + L] = min(dp[i + L], dp[i] + cost[r]);
        }
    }
    return dp[n] == INF ? -1 : (int)dp[n];
}
```

### [Q4. Minimum Number of String Groups Through Transformations](https://leetcode.com/problems/minimum-number-of-string-groups-through-transformations/)
`字串` `LC~est2150`

::: info 題意
給定字串陣列 words。對字串 s 定義變換:令 E 為偶數下標(0,2,4,…)字元組成的子序列、O 為奇數下標子序列;將 E 與 O 各自獨立向右循環位移任意量(可為 0),再放回原偶/奇位置重組。若 s 能經『一次』變換得到 t,則 s、t 等價。把 words 分成最少組,使同組任兩字串等價,求組數。約束:words.length≤1e5,∑|words[i]|≤5e5。

例:["abc","cab","bac","acb","bca","cba"]\(len3,E 取下標 0,2、O 取下標 1)。標準形:abc→(ac,b)、cba→(ac,b)、cab→(bc,a)、bac→(bc,a)、acb→(ab,c)、bca→(ab,c),共 3 種相異對 → 3。
:::

::: tip 作法
循環位移構成群(可組合、可逆),故等價 ⟺ **長度相同,且 E 互為循環旋轉、O 互為循環旋轉**——一次變換已涵蓋 (旋轉 E, 旋轉 O) 的所有組合,自動是等價關係。取每個字串的標準形 **(E 的最小循環表示, O 的最小循環表示)**,答案 = 相異標準形對數。用 Booth 演算法(或 Duval)O(m) 求最小旋轉,總計 O(∑|word|);(E,O) 各自的長度已隱含編碼原長,故不同長度不會撞 key。

```cpp
int leastRot(const string& s) {  // Booth O(n)
    int n = s.size();
    string ss = s + s;
    vector<int> f(2 * n, -1);
    int k = 0;
    for (int j = 1; j < 2 * n; j++) {
        int i = f[j - k - 1];
        while (i != -1 && ss[j] != ss[k + i + 1]) {
            if (ss[j] < ss[k + i + 1]) k = j - i - 1;
            i = f[i];
        }
        if (ss[j] != ss[k + i + 1]) {
            if (ss[j] < ss[k]) k = j;
            f[j - k] = -1;
        } else
            f[j - k] = i + 1;
    }
    return k;
}
string canon(const string& s) {
    if (s.empty()) return "";
    string ss = s + s;
    return ss.substr(leastRot(s), s.size());
}
int minGroups(vector<string>& words) {
    unordered_set<string> keys;
    for (auto& w : words) {
        string E, O;
        for (int i = 0; i < (int)w.size(); i++) (i & 1 ? O : E) += w[i];
        keys.insert(canon(E) + '|' + canon(O));  // '|'>'z',安全分隔
    }
    return keys.size();
}
```

> **Q3** [Transform Binary String Using Subsequence Sort](https://leetcode.com/problems/transform-binary-string-using-subsequence-sort/) `LC~est1850` — 「子序列排序只把 1 右移」的支配序不變量,IM 常見(掃過,無新亮點)
