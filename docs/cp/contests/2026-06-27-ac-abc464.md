# AtCoder Beginner Contest 464 (Japanese Student Championship 2026 Advance Qual) — 學習重點

> 懶人包:每題「技巧 + 作法」。各難度都收主流可遷移的;偏門/太難略過。難度 `~` 為估計,據官方 editorial 整理。

- **[A. Decisive Battle](https://atcoder.jp/contests/abc464/tasks/abc464_a)** `模擬` `~灰`
  作法:E 記 +1、W 記 -1,掃完看符號決定多數派。**長度奇數保證不平手**,只留「誰多」一個 bit 即可。

- **[B. Crop](https://atcoder.jp/contests/abc464/tasks/abc464_b)** `模擬` `~灰`
  作法:**直接掃出所有黑格的 min/max 行列得 bounding box**,輸出該子矩形即可,不必真的逐次刪行刪列(另解是寫 rotate 函式刪頂行轉 4 次)。

- **[C. Plumage Palette](https://atcoder.jp/contests/abc464/tasks/abc464_c)** `差分` `~灰`
  作法:每隻鳥拆成「初始 A 色 +1、第 D 天 A 色 -1、第 D 天 B 色 +1」三個事件,按天 bucket。**維護每色計數 cnt 與「種類數 kind」,0→1 時 kind++、1→0 時 kind--**,逐天輸出 kind,O(N+M)。

- **[D. Celester](https://atcoder.jp/contests/abc464/tasks/abc464_d)** `dp` `狀壓` `~棕`
  作法:逐日決定晴/雨,嬉しさ只取決於前一天天氣 → **只需保留「前一天=晴」「前一天=雨」兩個狀態的最佳值**。晴↔雨四種轉移,改變天氣扣對應 cost,O(N) 滾動 DP。

- **[E. Fill-Rect Query](https://atcoder.jp/contests/abc464/tasks/abc464_e)** `差分` `離線` `~綠`
  作法:**用操作序號 i(而非值)覆寫,每格答案=右下矩形內最大的 i**;從右下往左上做二維 max 傳播(2D imos 反向),最後把 i 還原成 X_i。另解:逆序處理只覆寫尚未被寫過的格子,均攤 O(HW+Q)。

- **[F. Random Vault Heist](https://atcoder.jp/contests/abc464/tasks/abc464_f)** `機率期望` `MITM` `組合計數` `~黃`
  作法:**「隨機選到違反條件為止」改看成「隨機排成一列、取前綴」**,到達狀態 S 的機率 = 1/C(N,|S|);答案 = Σ_{ΣA<X} (1/C(N,|S|))·(剩餘平均金額)。同 |S|=k 分母相同,故對每個 k 算「合法集合數」與「ΣA 總和」→ **折半枚舉(MITM)**:右半按大小 k 排序 + prefix sum,左半枚舉後雙指針/二分合併,O(N·2^{N/2})。

- **[G. Celester 2](https://atcoder.jp/contests/abc464/tasks/abc464_g)** `構造` `反悔貪心` `~橙`
  作法:**前後補 S…R,轉成相鄰異同的 01 列;SR 列改一字 = 01 列改相鄰兩位,讓 1 多 2 個等價於「把兩個 0 配對」**。歸約成:在「相鄰 0 距離陣列 D」中選 k/2 個、不可選相鄰、和最小 → 正是 JOI Candies 反悔貪心(選 D_i 後把 D_{i-1},D_i,D_{i+1} 併成 D_{i-1}+D_{i+1}-D_i),priority_queue + 雙向鏈表 O(N log N)(cost 和 ≤N,桶排可做到線性)。
