### [C. Plumage Palette](https://atcoder.jp/contests/abc464/tasks/abc464_c)
`差分` `~800`

::: info 題意
給定 N 隻鳥與 M 天觀察(N, M ≤ 3×10^5),每隻鳥 i 以三元組 (A_i, D_i, B_i) 描述:第 D_i − 1 天(含)以前是顏色 A_i、自第 D_i 天(含)起是顏色 B_i(顏色編號 1…N,允許 A_i = B_i 即不變色)。

輸出:M 行,第 j 行為第 j 天全體 N 隻鳥身上相異顏色的數量。

例:N=2, M=2,鳥 1 = (1,2,2)、鳥 2 = (2,2,2) → 第 1 天顏色為 {1,2} 輸出 2;第 2 天兩隻都是顏色 2,輸出 1。
:::

::: tip 作法
每隻鳥拆成三個事件(day 1:A 色 +1;day D_i:A 色 −1、B 色 +1)按天 bucket;**維護每色 cnt 與相異數 kind,0→1 時 kind++、1→0 時 kind--**,逐天輸出,O(N+M)。

```cpp
// sketch:事件差分 + kind 維護(ev[d] 收「第 d 天生效」的 (顏色, ±1))
for (auto [A, D, B] : birds) {
    ev[1].push_back({A, +1});
    ev[D].push_back({A, -1});
    ev[D].push_back({B, +1});
}
int kind = 0;
for (int d = 1; d <= M; ++d) {
    for (auto [c, dt] : ev[d]) {
        cnt[c] += dt;
        if (dt > 0 && cnt[c] == 1) kind++;   // 0 -> 1
        if (dt < 0 && cnt[c] == 0) kind--;   // 1 -> 0
    }
    cout << kind << '\n';
}
```

### [D. Celester](https://atcoder.jp/contests/abc464/tasks/abc464_d)
`dp` `狀壓` `~1200`

::: info 題意
給定長度 N 的天氣字串 S(`S` 晴、`R` 雨)與整數列 X_1…X_N、Y_1…Y_{N−1}(皆 ≤ 10^9):可任選若干天翻轉其天氣(晴↔雨),翻第 i 天花費 X_i;最終每個「第 i 天雨且第 i+1 天晴」的位置獲得 Y_i。

輸出:操作後可達到的最大淨快樂值(ΣY 收益 − ΣX 花費);多測,ΣN ≤ 2×10^5。

例:N=2, S=`SR`, X=[1,1], Y=[5] → 翻兩天成 `RS`:得 Y_1=5、花 1+1=2,淨 3;不翻是 0、只翻一天是 −1 → 答案 3。
:::

::: tip 作法
相鄰貢獻只取決於連續兩天 → **只保留「前一天=晴」「前一天=雨」兩狀態**的滾動 DP,四種轉移各扣翻轉 cost,O(N)。

```cpp
// sketch:兩狀態滾動 DP(0=晴 S,1=雨 R)
long long dp[2];
dp[0] = (s[1] == 'S') ? 0 : -x[1];
dp[1] = (s[1] == 'R') ? 0 : -x[1];
for (int i = 2; i <= n; ++i) {
    long long c0 = (s[i] == 'S') ? 0 : -x[i];         // 今天定為晴
    long long c1 = (s[i] == 'R') ? 0 : -x[i];         // 今天定為雨
    long long nd0 = max(dp[0], dp[1] + y[i-1]) + c0;  // 昨雨今晴 → 收 Y
    long long nd1 = max(dp[0], dp[1]) + c1;
    dp[0] = nd0; dp[1] = nd1;
}
long long ans = max(dp[0], dp[1]);
```

### [E. Fill-Rect Query](https://atcoder.jp/contests/abc464/tasks/abc464_e)
`差分` `離線` `~1500`

::: info 題意
給定 H×W 網格(H×W ≤ 10^6),初始每格都寫著字母 `A`。

依序執行 Q ≤ 2×10^5 次操作,第 i 次給 (R_i, C_i, X_i):把左上角 (1,1)、右下角 (R_i, C_i) 的矩形內所有格子覆寫成大寫字母 X_i。

輸出:所有操作結束後的盤面,共 H 行、每行 W 個字母。

例:H=W=2,依序 (R,C,X) = (1,2,`B`)、(2,1,`C`) → 格 (1,1) 被蓋兩次取較晚的 `C` → 最終盤面 `CB` / `CA`。
:::

::: tip 作法
**用操作序號 i(而非字母)覆寫:每格答案 = 滿足 R_i≥r、C_i≥c 的最大 i**——把 i 放到 (R_i, C_i),從右下往左上做二維後綴 max,最後把 i 還原成 X_i。

另解:逆序處理、只填尚未寫過的格子,均攤 O(HW+Q)。

```cpp
// sketch:序號覆寫 + 右下往左上二維後綴 max(id 開 (H+2)×(W+2) 全 0)
for (int i = 1; i <= Q; ++i) id[R[i]][C[i]] = i;   // 同格較大 i 自動蓋掉
for (int r = H; r >= 1; --r)
    for (int c = W; c >= 1; --c)
        id[r][c] = max({id[r][c], id[r+1][c], id[r][c+1]});
// 還原:id[r][c] == 0 ? 'A' : X[id[r][c]]
```

### [F. Random Vault Heist](https://atcoder.jp/contests/abc464/tasks/abc464_f)
`機率期望` `MITM` `組合計數` `~2600`

::: info 題意
給定 N ≤ 40 個保險箱的金額 A_1…A_N(A_i ≤ 10^16)與門檻 X(1 ≤ X ≤ ΣA_i)。

搶匪反覆執行「從尚未開過的箱子中等機率隨機選一個,打開並拿走裡面全部的錢」,直到偷得總額 ≥ X 為止停手。

輸出:偷得總額的期望值 mod 998244353(有理數取模)。

例:N=2, A=[1,2], X=2 → 先開箱 1(機率 1/2)未達門檻再開箱 2,共拿 3;先開箱 2 直接停,拿 2 → 期望 (3+2)/2 = 5/2(公式驗:S=∅ 貢獻 3/2、S={1} 貢獻 2/C(2,1)=1,合計 5/2)。
:::

::: tip 作法
**「隨機開到停」改看成「隨機排成一列、取前綴」**:途中經過集合 S 的機率 = 1/C(N,|S|),答案 = Σ_{ΣA(S)<X} (剩餘平均金額)/C(N,|S|);同 |S|=k 分母相同,只需每個 k 的「合法集合數 + ΣA 總和」→ **折半枚舉**:右半按 (大小, 和) 排序 + prefix sum,左半枚舉後二分/雙指針合併,O(N·2^{N/2})。

```cpp
// sketch:折半 —— 對每個 k 求 cnt[k]、sum[k](合法 S:ΣA(S) < X)
// 右半 2^{N/2} 子集按 (大小, 和) 排序,備好 (個數, 和) 前綴和
for (auto [szL, sumL] : lefts)                       // 枚舉左半子集
    for (int szR = 0; szR <= nR; ++szR) {
        auto [c, s] = prefix_below(szR, X - sumL);   // 和 < X - sumL 者
        cnt[szL + szR] += c;
        sum[szL + szR] += s + (__int128)c * sumL;    // 實作時邊加邊取模
    }
// 每個 k 的貢獻 = (cnt[k]·ΣA − sum[k]) / (N−k) / C(N,k)
```
:::

### [G. Celester 2](https://atcoder.jp/contests/abc464/tasks/abc464_g)
`構造` `反悔貪心` `~3000`

::: info 題意
給定長度 N(ΣN ≤ 10^6)的天氣字串 S(`S` 晴、`R` 雨),一次操作可把某一天的天氣晴↔雨互換。

對每個 k = 0, 1, …, N 問:至多操作 k 次後,「第 i 天雨且第 i+1 天晴」的位置對數最大值。

輸出:一行 N+1 個整數 A_0 A_1 … A_N(A_k 為至多 k 次操作的答案);多測。

例:N=2, S=`SR` → k=0:`SR` 沒有「雨→晴」對;k=1:翻成 `RR`/`SS` 也都是 0;k=2:翻成 `RS` 得 1 → 輸出 `0 0 1`。
:::

::: tip 作法
**前後補 S…R,轉成相鄰異同的 01 列;改一字 = 翻相鄰兩位,答案 +1 等價於「把兩個 0 配對」,配對距離 D 的一對花 D 次翻轉**。

歸約成:對每個 j 求「相鄰 0 距離陣列 D 中選 j 個、不可選相鄰、和最小」——正是 JOI Candies 反悔貪心(選 D_i 後把 D_{i-1}, D_i, D_{i+1} 併成 D_{i-1}+D_{i+1}−D_i),priority_queue + 雙向鏈表 O(N log N)(cost 和 ≤ N,桶排可做到線性)。

```cpp
// sketch:反悔貪心主迴圈(JOI Candies 型;PLI = pair<long long,int>)
priority_queue<PLI, vector<PLI>, greater<PLI>> pq;  // (距離 D_i, 節點 i)
long long cost = 0;
for (int j = 1; j <= tot / 2; ++j) {      // 第 j 對
    auto [d, i] = pop_valid(pq);          // 取最小,懶刪已失效節點
    cost += d;                            // 湊滿 j 對的最小總翻轉數
    int l = L[i], r = R[i];
    D[i] = D[l] + D[r] - d;               // 反悔:退掉 i、改選左右兩鄰
    erase(l); erase(r);                   // 雙向鏈表摘除 + 標失效
    pq.push({D[i], i});
    // 所有 k ≥ cost 的答案自此輪起再 +1
}
```
