### [A. Another Puzzle from Papyrus](https://codeforces.com/contest/2238/problem/A)
`實作` `~800`

::: info 題意
給定兩個長度 n 的整數陣列 a、b 與整數 c(n、c、值皆 ≤ 100)。每次操作可任選 i 把 aᵢ 減 1(花 1 秒),或將 a 任意重排(花 c 秒)。

求/輸出:把 a 變成 b 的最少總秒數;不可能則 −1。

例:`a=[2,1], b=[1,2], c=1` → 原序 a₂=1<b₂=2 只能重排:重排成 [1,2] 花 1 秒即等於 b,Σ(aᵢ−bᵢ)=0 不需再減 → 答案 1。
:::

::: tip 作法
減量總和固定 = **Σ(aᵢ−bᵢ)**;原序若有 aᵢ<bᵢ 則加一次 c、兩排各自排序後重驗,仍有 aᵢ<bᵢ 輸出 −1。

純實作,熱身略過。
:::

### [B. Crimson Triples](https://codeforces.com/contest/2238/problem/B)
`數論` `貢獻` `~1100`

::: info 題意
給定整數 n(n≤2×10⁵)。有序三元組 (a,b,c) 滿足 1≤a,b,c≤n(可重複),若 gcd(lcm(a,b), lcm(b,c)) = gcd(a,c) 則稱為 crimson。

求/輸出:crimson 三元組的總數;兩三元組只要任一位置的值不同即視為不同。

例:`n=2` → 全部 8 組窮舉,恰 (1,1,1)、(1,1,2)、(2,1,1)、(2,1,2)、(2,2,2) 這 5 組滿足(如 (2,1,2):gcd(lcm 2, lcm 2)=2=gcd(2,2) ✓;(1,2,1):gcd(2,2)=2≠1=gcd(1,1) ✗)→ 答案 5 = ⌊2/1⌋²+⌊2/2⌋²。
:::

::: tip 作法
這種怪條件的題,第一步是猜出乾淨的等價形式(寫個小 n 暴力對照最快)。這裡的等價是:**條件成立 ⟺ b 同時整除 a 與 c**。

兩個方向都短:若 b|a 且 b|c,則 lcm(a,b)=a、lcm(b,c)=c,左式直接就是 gcd(a,c) ✓。反過來,lcm(a,b) 和 lcm(b,c) 都是 b 的倍數,所以左式 gcd 也是 b 的倍數;條件說左式 = gcd(a,c),於是 b | gcd(a,c),即 b|a 且 b|c。

化簡完計數就是送分:固定 b,a 與 c 各自要是 b 的倍數且 ≤ n,各 ⌊n/b⌋ 種,答案 = **Σ_{b=1}^{n} ⌊n/b⌋²**。直接算 O(n) 就過,值域更大時上整除分塊 O(√n)。
:::

### [C. Village Guilds](https://codeforces.com/contest/2238/problem/C)
`樹` `dp` `~1400`

::: info 題意
給定一棵 n 個點、以 1 為根的有根樹(n≤2×10⁵)。對每個點 v 與非負整數 h,「公會」定義為 v 的子樹中與 v 距離恰為 h 的點所成的集合(h=0 時即 {v}),兩公會不同若且唯若集合不同。

求/輸出:整棵樹中相異非空公會的個數。

例:根 1 有子 2、3,3 有子 4 → 全部公會:{1}、{2,3}、{2}、{3}、{4}({4} 由 v=1,h=2、v=3,h=1、v=4,h=0 三處產生,只計一次)→ 答案 5。
:::

::: tip 作法
麻煩在去重:同一個集合會被很多組 (v, h) 生成,像例子裡的 {4}。要找一個規則,讓每個集合只在一個地方被算。

觀察 (v, h) 的公會什麼時候「不是新的」:如果 h 超過 v 的**次深分支**能到的深度,那一層的點全部落在最深分支裡——這個集合跟(最深孩子, h−1)的公會一模一樣,輪到那個孩子時自然會算到。反過來 h 還在次深分支範圍內時,該層至少橫跨兩個分支,集合的 LCA 就是 v 本人——**這種集合只有 v 生得出來**,不會跟任何別的 (v', h') 重複。

所以每個點的貢獻 = 「次深分支深度 − depth(v) + 1」(h 從 0 到次深都各給一個新公會;h=0 的 {v} 也含在內)。一次 DFS 對每個點維護子分支 top-2 深度,O(n)。另一個等價視角:逐層把同層點按 DFS 序排,數相鄰點的相異 LCA。

```cpp
// sketch:一次 DFS,維護子分支 top-2 最深(絕對)深度
int dfs(int v, int d) {                    // 回傳子樹內最大深度
    int mx1 = d, mx2 = d;                  // 最深、次深(缺的分支以 d 計)
    for (int u : g[v]) {
        int dn = dfs(u, d + 1);
        if (dn > mx1) { mx2 = mx1; mx1 = dn; }
        else if (dn > mx2) mx2 = dn;
    }
    ans += mx2 - d + 1;                    // h = 0..(mx2-d) 各是一個新公會
    return mx1;
}
```

### [D. Storming Arasaka](https://codeforces.com/contest/2238/problem/D)
`數論` `構造` `~1700`

::: info 題意
每筆詢問給定整數 n(n≤10⁶,t≤10⁴)。把 n 的所有大於 1 的因數(含 n 本身)分成若干非空有序層 L₁,…,L_k,須滿足:任一數 x 的因數(1 與 x 自身除外)全落在比 x 所在層更早的層,且每層內所有數能排成一條相鄰兩數 gcd>1 的鏈。

求/輸出:滿足條件的最少層數 k。

例:`n=6` → 因數 {2,3,6}:2、3 同層須相鄰但 gcd(2,3)=1 不成鏈,只能各佔一層;6 的因數 2、3 又須在更早層 → 最少 3 層,如 [2],[3],[6]。
:::

::: tip 作法
答案 = **Ω(n) + ω(n) − 1**(Ω 計重數的質因數個數、ω 相異質因數個數)。這種閉式答案,重點是下界跟構造各自怎麼來。

**下界有兩股**。第一股:取一條因數鏈 p → p² → … → n(每步乘一個質因數,長 Ω),鏈上任兩個有整除關係、規則逼它們分屬不同層 → 至少 Ω 層。第二股:**含質數 p 的層必為單元素層**——跟 p 同層又要 gcd>1,只能是 p 的倍數,但 p 的倍數以 p 為真因數、被規則趕去更晚的層,矛盾;所以 ω 個質數各霸佔一層。兩股重疊一層(因數鏈的起點也是質數層),合計至少 Ω + ω − 1。

**構造達成**:質數各佔一層(ω 層),其餘合數按 Ω 值分層(Ω=2 到 Ω(n) 共 Ω−1 層);同層的合數 Ω 相同、無整除關係,按質因數排序可串成相鄰 gcd>1 的鏈(如 n=30 的 Ω=2 層 {6,10,15}:6–10 共享 2、10–15 共享 5)。線性篩預處理 Ω、ω 後每筆詢問 O(1)。驗算:n=120=2³·3·5 → 5+3−1=7。
:::

### [E. Cake Trial](https://codeforces.com/contest/2238/problem/E)
`dp` `狀態擴充` `~2000`

::: info 題意
給定長度 n 的字串 s,字元為 T(真)/F(假)/N(未定)(n≤500,Σn³≤500³)。

對手先把每個 N 填成 T 或 F;你看到最終字串後,選一段連續區間(可空)宣稱「區間內全 F、區間外全 T」,錯誤數 = 區間外的 F 數 + 區間內的 T 數,你會選錯誤數最少的區間。

求/輸出:對手以最大化你的最少錯誤數為目標填 N,求該最大值。
:::

::: tip 作法
先解內層「固定盤面你怎麼選」:錯誤數 = 區間外 F + 區間內 T = #F −(區間內 F − 區間內 T),所以最少錯誤 = **#F − 最大子段和**(F 記 +1、T 記 −1,允許空段)。內層有了閉式,外層就是對手在最大化「#F − max 子段和」。

麻煩在這兩個量都取決於整個填法,不能分開貪。解法是**把 Kadane 跑到一半的內部狀態直接搬進 DP 維度**:掃到位置 i 時,Kadane 只記兩個數——clamp 在 0 的現值 cur、與歷史最大。令 dp[i][f][c] = 前 i 位填了 f 個 F、現值為 c 時,**最小可達的歷史最大值**(歷史值單調不減,適合當「值」而不是維度)。每位試填 F(+1)或 T(−1、clamp),答案 = max(f − dp[n][f][c])。O(n³),Σn³ 有保。

這題值得記的是手法本身:**演算法(Kadane)的執行狀態也可以是 DP 狀態**——對手在「對抗一個演算法」時,把該演算法的內部變數建進狀態就對了。另解:枚舉子段和上界 D、由左往右貪心壓 T 數,O(n²)。

```cpp
// sketch:dp[i][f][c] = 前 i 位、f 個 F、Kadane 現值 c(clamp 在 0)時
//        最小可達的「歷史最大子段和」;對手要讓 f - 歷史值 最大
for (int i = 0; i < n; ++i)
  for (int f = 0; f <= i; ++f)
    for (int c = 0; c <= i; ++c) if (dp[i][f][c] < INF) {
      int cur = dp[i][f][c];
      if (s[i] != 'T')   // 這位填 F(+1)
        chmin(dp[i+1][f+1][c+1], max(cur, c + 1));
      if (s[i] != 'F')   // 這位填 T(-1)
        chmin(dp[i+1][f][max(c - 1, 0)], cur);
    }
// ans = max over (f, c) of f - dp[n][f][c]
```
:::

### [F. Infinite Work](https://codeforces.com/contest/2238/problem/F)
`組合計數` `反射原理` `~2300`

::: info 題意
近乎無限的完全 binary tree(1 為根,點 i 的兒子為 2i、2i+1),初始所有點皆在職。

每天兩階段:先「補雇」——每個在職點讓其不在職的直接兒子復職(當天新復職者不再雇人);再「開除」——任選一組在職點直接開除,被除者整棵子樹連鎖被開除(1 不可除,且每點終生至多被開除一次)。給定 n, k(n≤10⁹,k≤2×10⁵,t≤10⁵)。

求/輸出:n 天後恰剩 k 個在職點、且直接開除總次數最少的開除方案數 mod 1e9+7。

例:`n=1, k=2` → 最終在職集須含根且向上封閉,大小 2 只有 {1,2} 與 {1,3};兩者都恰需 3 次直接開除(留 {1,2} 就開除 3、4、5 三棵子樹)→ 答案 2。
:::

::: tip 作法
先看最終狀態長什麼樣:剩下的在職集是**含根的連通塊**,而 k 個點的連通塊在 binary tree 上恰有 **k+1 棵懸掛子樹**被砍掉(k 個點共 2k 個兒子位,內部用掉 k−1 個,剩 k+1 個缺口)——這就是 K = k+1 的來源,也解釋了為何最少直接開除次數跟砍掉的懸掛塊結構綁在一起。

天數 n 的限制體現在「懸掛塊能掛多淺」:補雇會把砍淺的位置重新長回來,深度太淺(對應 K 的二進位中位 ≥ n 的位)的懸掛塊撐不完 n 天,得**把該位反覆「拆成兩個低一位」**壓到位 < n(允許同一位出現多次)。拆完得到 sm 個 2 的冪、位 i 出現 bits[i] 次。

方案計數 = 「以這組 2 的冪為葉子的相異 binary tree 形」數:樹形本身是 **Catalan(sm−1)**(反射原理閉式 C(2m,m) − C(2m,m−1)),葉子是多重集所以再乘 **sm!/∏ bits[i]!** 去重。預處理階乘與逆階乘後每筆詢問 O(log)。(作法據參考 code 重建,拆位與計數的對應以 code 行為為準。)
:::
