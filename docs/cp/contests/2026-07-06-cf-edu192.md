### [B. Predominant Frequency Division](https://codeforces.com/contest/2242/problem/B)
`貪心` `分類討論` `~1200`

::: info 題意
給定長度 n 的陣列 a,每個元素都是 1、2 或 3(Σn ≤ 2e5)。

問能否把 a 切成三段連續非空子陣列,使第 i 段(i=1..3)中「值 > i 的元素個數」不超過該段長度的一半;等價地:第 1 段中 1 的個數 ≥ 2、3 的總數,第 2 段中 1、2 的總數 ≥ 3 的個數,第 3 段恆合法、只需非空。

輸出:可行則 YES,否則 NO。

例:a = [1, 2, 3] 切 [1] | [2] | [3]:段 1 有一個 1 ≥ 零個 2、3,段 2 的 1、2 共一個 ≥ 零個 3,段 3 非空 → YES;a = [3, 3, 3] 唯一切法是 [3] | [3] | [3],段 1 中值 > 1 的個數 1 超過長度一半 0.5 → NO。
:::

::: tip 作法
先把三個條件各自看清楚。第 3 段的條件是「值 > 3 的個數不超過一半」——但值域只有 1..3,值 > 3 根本不存在,所以**第 3 段永遠合法,只要非空**。剩下兩段都是「多數檢查」:第 1 段要 1 至少占一半(1 的個數 ≥ 2、3 的總數),第 2 段要 3 至多占一半(1、2 總數 ≥ 3 的個數)。

多數檢查的標準寫法是 **±1 前綴和**:第 1 段把 1 記 +1、其餘記 −1,合法 ⟺ 段內和 ≥ 0;第 2 段把 3 記 −1、其餘 +1,同樣是段內和 ≥ 0。段內和可以寫成前綴差,所以固定兩個切點——第 1 段結尾 j、第 2 段結尾 i(i < n,保留第 3 段非空)——條件就是 pref1[j] ≥ 0 且 pref2[i] − pref2[j] ≥ 0。

一次掃描搞定:掃 i 的同時,對所有已確認合法(pref1[j] ≥ 0)的 j 維護 pref2[j] 的最小值;只要 pref2[i] ≥ 這個最小值就 YES。O(n)。

```cpp
// sketch:單迴圈掃第 2 段結尾 i(i < n,保留第 3 段非空)
int p1 = 0, p2 = 0, mn = INT_MAX;
bool ok = false;
for (int i = 1; i <= n - 1; ++i) {
    p2 += (a[i] == 3 ? -1 : +1);      // pref2:3 記 −1、其餘 +1
    if (p2 >= mn) ok = true;          // 存在合法第 1 段結尾 j < i
    p1 += (a[i] == 1 ? +1 : -1);      // pref1:1 記 +1、其餘 −1
    if (p1 >= 0) mn = min(mn, p2);    // j = i 之後可作第 1 段結尾
}
```

### [C. Unstable Elements](https://codeforces.com/contest/2242/problem/C)
`數學` `~1400`

::: info 題意
給定已排序的整數陣列 a(長度 n)與目標長度 k(n, k ≤ 3e5)。

每次操作先標記每個等值連續段的首元素(即 a_1 與所有 a_i ≠ a_{i−1} 的位置),再二選一:把標記元素全部刪除,或把每個標記元素換成兩個相同元素;不允許把陣列操作到空。

輸出:經任意次操作可到達、長度恰為 k 的相異陣列個數。

例:a = [1, 2, 2]、k = 2 → 標記位置 1、2:刪除得 [2],再對 [2] 做「換成兩個」得 [2, 2],長度 2 ✓;可驗證長度 2 的可達陣列僅 [2, 2] 一個,答案 1。
:::

::: tip 作法
先換一個視角:陣列已排序,把等值連續段壓成 block 長度序列。「標記每段首元素」在 block 視角下非常乾淨——**刪除 = 每個存活 block 長度同時 −1(長度歸零的整段消失),複製 = 每個存活 block 長度同時 +1**。整個過程就是所有 block 的長度同步升降,歸零者永久出局(且不允許全滅)。

接著問:最終陣列由什麼決定?值的相對順序不會變,所以只由「哪些 block 還活著」+「當前的同步偏移量」決定。而「哪些活著」又只取決於**歷史最低點**——過程中長度被壓到最低的那一刻,原長不夠的 block 都死了。所以枚舉門檻 i(過程中最短存活 block 的原長;需 c[i] > 0,門檻才真的踩得到):此時原長 ≥ i 的 block 都活著、數量 s1,壓到最低點時總長是 s2 = Σ(原長 − i + 1);之後只能全體 +1 拉高,總長每次增加 s1。

於是「長度 k 可達」⟺ k ≥ s2 且 (k − s2) 是 s1 的倍數;不同門檻對應不同存活集合、也就是不同陣列,合法門檻數即答案。由大到小枚舉 i,s1、s2 都能遞推,整體 O(n)。

```cpp
// sketch:c[i] = 原長恰為 i 的 block 數,由大到小枚舉門檻
long long s1 = 0, s2 = 0, ans = 0;   // 存活 block 數 / 最小可達總長
for (int i = n; i >= 1; --i) {
    s1 += c[i];
    s2 += s1;                         // 門檻 −1:每個存活 block 最小長 +1
    if (c[i] && k >= s2 && (k - s2) % s1 == 0) ans++;
}
```

### [D. Two Digit Strings](https://codeforces.com/contest/2242/problem/D)
`dp` `不變量` `字串` `~1700`

::: info 題意
給定兩個數字字串 a、b(Σ(|a|+|b|) ≤ 1e4)。

每次操作可在任一串中選相鄰兩個字元,替換成兩者之和 mod 10,該串長度 −1;兩串各可操作任意次(含零次)。

輸出:兩串能變得完全相等(長度與逐位皆同)時的最大長度;不可能則輸出 −1。

例:a = "12"、b = "21" → digit sum 皆 3;維持長度 2 就不能操作,"12" ≠ "21";各做一次操作得 "3" = "3" → 答案 1。
:::

::: tip 作法
操作把相鄰兩字元 a、b 換成 (a+b) mod 10,先找不變量:**整串的 digit sum mod 10 操作前後不變**。所以兩串總和 mod 10 不同,連長度 1 都湊不齊,直接 −1。

再想「變成同一個長 k 的串」的充要條件。目標串的每個字元,對應原串的一段連續區間(那段一路併成單字元),而併出來的值就是**段和 mod 10**。所以兩串能變成同一個長 k 串 ⟺ 各自能切成 k 段、對應段的和 mod 10 逐段相等。段和用前綴和表示後,這句話等價於:**兩串的前綴和(mod 10)序列有一組共同的切點值序列**(含開頭的 0 與結尾的總和)。

求最長共同切點序列,就是經典 **LCS**:對 pa、pb(各含空前綴的 0)做 LCS,k 段需要 k+1 個切點,答案 = LCS − 1。開頭的 0 天然對上,結尾靠不變量保證相等。一道看起來要做子段 DP 的題,就這樣壓成 O(nm) 的 LCS。

```cpp
// sketch:切點值 = 前綴和 mod 10(含空前綴的 0),對兩陣列做經典 LCS
vector<int> pa{0}, pb{0};
for (char c : a) pa.push_back((pa.back() + c - '0') % 10);
for (char c : b) pb.push_back((pb.back() + c - '0') % 10);
if (pa.back() != pb.back()) return -1;             // 不變量不同
for (int i = 1; i <= (int)pa.size(); ++i)
    for (int j = 1; j <= (int)pb.size(); ++j)
        dp[i][j] = pa[i-1] == pb[j-1] ? dp[i-1][j-1] + 1
                                      : max(dp[i-1][j], dp[i][j-1]);
return dp[pa.size()][pb.size()] - 1;               // 共同切點數 − 1 = 段數
```

### [E. Product of Closures](https://codeforces.com/contest/2242/problem/E)
`位元` `構造` `貪心` `~2400`

::: info 題意
定義 C(x) 為把正整數 x 的二進位表示(無前導零)無限重複串接成的無限 01 串,兩串的乘積 C(x) AND C(y) 為逐位 AND。

給定整數 l、r、n(l < r < 2^30,n ≤ 1000)。求一對 x、y 滿足 l ≤ x < y ≤ r,使 C(x) AND C(y) 字典序最小。

輸出:該乘積的前 n 個字元。

例:l=2, r=4, n=4 → C(2)=1010…、C(3)=1111…、C(4)=100100…;三對的 AND 前 4 字元:(2,3)→1010、(2,4)→1000、(3,4)→1001,字典序最小為 1000,取 x=2、y=4。
:::

::: tip 作法
直接枚舉所有 (x, y) 對不可行(區間可到 2^30 寬),得先把候選砍下來。**支配觀察:把 x 的最低位 1 清掉,若結果仍 ≥ l,那麼新數二進位長度不變、每一位都 ≤ 原來**——於是 C(新) 逐位 ≤ C(原)(週期一樣、逐位比較),不管搭配誰,AND 出來字典序都不會更差。換句話說原 x 被支配,可以直接丟掉。

把「清最低位 1」清到不能再清,活下來的極小候選都長同一個樣子:**l 的高位前綴 + 某個原本是 0 的位改成 1 + 更低位全清 0**(這是「≥ l 的前提下盡量清空低位」的邊界形),加上 l 本身,總共只有 O(log r) 個。

x、y 各自從候選裡枚舉(約 30 × 29 對),每對直接模擬 AND 的前 n 個字元(第 i 位 = x 串的第 i mod |x| 位 AND y 串的第 i mod |y| 位),取字典序最小,O(30² · n)。官解是 O(1) 的分類構造,但這條「從暴力出發、用支配剪枝壓候選」的路線好想得多,casework 全免。

```cpp
// sketch:候選生成 —— l 本身 + 「l 的前綴 + 某 0 位改 1 + 低位清 0」
vector<int> cand{l};
for (int b = 0; b < 30; ++b)
    if (!(l >> b & 1)) {
        int v = ((l >> b) | 1) << b;   // 高位照抄、第 b 位設 1、低位清 0
        if (v <= r) cand.push_back(v);
    }
// x、y 於 cand 中各自枚舉(x < y),每對 O(n) 生成 AND 前綴取字典序最小
```

### [F. Summer Vacation](https://codeforces.com/contest/2242/problem/F)
`資料結構` `根號` `~2500`

::: info 題意
給定 n 天(n ≤ 1e5)與每天的空閒分鐘數 a_i(a_i ≤ n)。

從 0 元開始依序過日子:某天開始時若持有錢 ≥ a_i,則整天消費、錢 −a_i;否則整天打工、錢 +a_i(每天只能整天擇一)。

輸出:對每個 k = 1..n,若只從第 n−k+1 天起(以 0 元)過最後 k 天,最後一天結束後剩多少錢——共 n 個答案。

例:a = [2, 1, 2] → k=1:0 < 2,打工得 2;k=2:兩天皆打工,0 → 1 → 3;k=3:0 → 2(打工)→ 1(2 ≥ 1,消費)→ 3(打工)。輸出 2 3 3。
:::

::: tip 作法
n 個後綴各自模擬是 O(n²),要找可以共用的結構。倒著疊:令 g_i(v) = 「從第 i 天開始、手上 v 元」走到最後剩的錢,g_{n+1} 是恆等函數,答案是每個 g_i(0)。先確認定義域有限:**錢永遠 < 2n**——打工發生的前提是 v < a_i ≤ n,打完 < 2n;消費只會變少。所以每個 g_i 都能存成一張長 2n 的表(位置 v 存 g_i(v))。

妙處在「疊上一天」是整段搬移,不用逐格算:v < a_i 的人打工,去查舊表的 v + a_i,也就是新表的 [0, a) 段整段等於**舊表 [a, 2a)**;v ≥ a_i 的人消費,查舊表 v − a_i,新表 [a, 2n) 段等於**舊表 [0, 2n−a)**。合起來:**新序列 = 舊 [a, 2a) 接 舊 [0, 2n−a)**,一次 cut-paste,長度不變:

序列的 cut-paste 用**持久化 treap** 的 split/merge 做(treap 基礎見技巧卡 [Treap](/cp/techniques/implicit-treap))(取段要共享結構,記憶體用週期性重建控制),或 sqrt 分塊;每輪答案就是當前序列的第 0 格。另外 1.5s 時限下,O(n²) 交給編譯器自動向量化的暴力其實也能過——先寫暴力驗對、再決定要不要上資料結構,是實戰上比較穩的順序。

```cpp
// sketch:seq 初始為恆等 [0..2n),倒序疊天(range 取段需持久化)
for (int i = n; i >= 1; --i) {
    int a = ai[i];
    Node *mid = range(seq, a, 2*a);      // 舊 [a, 2a):v < a 打工 → 查 v+a
    Node *low = range(seq, 0, 2*n - a);  // 舊 [0, 2n−a):v ≥ a 消費 → 查 v−a
    seq = merge(mid, low);               // cut-paste,長度仍 2n
    ans[n - i + 1] = kth(seq, 0);        // 初始錢 0 → 最後 k = n−i+1 天的答案
}
```
