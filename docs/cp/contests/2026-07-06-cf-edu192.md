# Educational Codeforces Round 192 — 學習重點

> 懶人包:每題「題意 + 核心作法」。難度 `~` 為 CF-equivalent。

- **[B. Predominant Frequency Division](https://codeforces.com/contest/2242/problem/B)** `貪心` `分類討論` `~1200`
  題意:給定長度 n 的陣列 a,每個元素都是 1、2 或 3(Σn ≤ 2e5)。問能否把 a 切成三段連續非空子陣列,
  使第 i 段(i=1..3)中「值 > i 的元素個數」不超過該段長度的一半;等價地:第 1 段中 1 的個數 ≥ 2、3 的總數,
  第 2 段中 1、2 的總數 ≥ 3 的個數,第 3 段恆合法、只需非空。輸出:可行則 YES,否則 NO。
  作法:**第 3 段無值 >3、永遠合法**,前兩段條件各是一個 ±1 前綴和 ≥ 0(1 段:1 記 +1 其餘 −1;2 段:3 記 −1 其餘 +1);掃第 2 段結尾 i < n,對已見的合法第 1 段結尾 j 維護 pref2[j] 最小值,pref2[i] ≥ min 即 YES。O(n)。

- **[C. Unstable Elements](https://codeforces.com/contest/2242/problem/C)** `數學` `~1400`
  題意:給定已排序的整數陣列 a(長度 n)與目標長度 k(n, k ≤ 3e5)。每次操作先標記每個等值連續段的首元素
  (即 a_1 與所有 a_i ≠ a_{i−1} 的位置),再二選一:把標記元素全部刪除,或把每個標記元素換成兩個相同元素;
  不允許把陣列操作到空。輸出:經任意次操作可到達、長度恰為 k 的相異陣列個數。
  作法:壓成 block 長度後,**每次操作 = 所有存活 block 長度同時 ±1,歸零者永久消失**,故可達陣列由「最短存活 block 的原長門檻」唯一決定。由大到小枚舉門檻 i(需 c[i] > 0),維護存活數 s1 與最小總長 s2(s1 += c[i]、s2 += s1);k ≥ s2 且 (k−s2) % s1 == 0 則答案 +1。O(n)。

- **[D. Two Digit Strings](https://codeforces.com/contest/2242/problem/D)** `dp` `不變量` `字串` `~1700`
  題意:給定兩個數字字串 a、b(Σ(|a|+|b|) ≤ 1e4)。每次操作可在任一串中選相鄰兩個字元,替換成兩者之和 mod 10,
  該串長度 −1;兩串各可操作任意次(含零次)。輸出:兩串能變得完全相等(長度與逐位皆同)時的最大長度;
  不可能則輸出 −1。
  作法:**digit sum mod 10 是不變量**(總和不同直接 −1);變成長 k 相等 ⟺ 兩串各切 k 段、對應段和 mod 10 相等 ⟺ 找兩串前綴和 (mod 10) 陣列的共同切點 → 直接做 LCS,答案 = LCS − 1。看似子段 DP 的題壓成 O(nm)。

- **[E. Product of Closures](https://codeforces.com/contest/2242/problem/E)** `位元` `構造` `貪心` `~2400`
  題意:定義 C(x) 為把正整數 x 的二進位表示(無前導零)無限重複串接成的無限 01 串,兩串的乘積 C(x) AND C(y)
  為逐位 AND。給定整數 l、r、n(l < r < 2^30,n ≤ 1000)。求一對 x、y 滿足 l ≤ x < y ≤ r,
  使 C(x) AND C(y) 字典序最小。輸出:該乘積的前 n 個字元。
  作法:**支配觀察:把 x 的最低位 1 清掉後若仍 ≥ l,新數每一位都 ≤ 原數,原 x 可捨** → 候選只剩「l 的前綴 + 某個 0 位改 1 + 低位全清 0」的 O(log r) 個;x、y 各自枚舉(約 30×29 對),每對 O(n) 求 AND 取最小。官解是 O(1) 分類構造;此法從暴力直接優化可得,省去繁瑣 casework。

- **[F. Summer Vacation](https://codeforces.com/contest/2242/problem/F)** `資料結構` `根號` `~2500`
  題意:給定 n 天(n ≤ 1e5)與每天的空閒分鐘數 a_i(a_i ≤ n)。從 0 元開始依序過日子:某天開始時若持有錢 ≥ a_i,
  則整天消費、錢 −a_i;否則整天打工、錢 +a_i(每天只能整天擇一)。輸出:對每個 k = 1..n,
  若只從第 n−k+1 天起(以 0 元)過最後 k 天,最後一天結束後剩多少錢——共 n 個答案。
  作法:**倒序加天數,把後綴的複合轉移函數存成長 2n 的序列——位置 v 存「初始錢 v 的最終錢」(錢恆 < 2n)**;疊上一天 = 序列 cut-paste(新序列 = 舊 [a_i, 2a_i) 接 [0, 2n−a_i)),答案 = 首元素。持久化 treap split/merge(週期重建控記憶體)或 sqrt 分塊;1.5s 時限下 O(n²) 自動向量化暴力也能過。
