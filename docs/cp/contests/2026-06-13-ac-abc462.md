### [C. Not Covered Points](https://atcoder.jp/contests/abc462/tasks/abc462_c)
`Pareto` `~700`

::: info 題意
給定二維平面上 N 個點(N≤3×10^5),點 i 座標為 (X_i,Y_i),保證 X、Y 各自為 1..N 的排列。對每個 i,考慮以 (0,0) 為左下角、(X_i,Y_i) 為右上角、邊平行於座標軸的矩形。

求:矩形內部(不含邊界)不包含 N 點中任何一點的 i 之個數,輸出一個整數。

例:`N=3`,點 `(1,2), (2,1), (3,3)` → `(1,2)`、`(2,1)` 的矩形內部皆無點;`(3,3)` 的矩形內部含另兩點 → 答案 `2`。
:::

::: tip 作法
點 i 合法 ⟺ 所有 X_j<X_i 的 j 都有 Y_j>Y_i;**依 X 升序排序後即「Y 是前綴嚴格最小值」的點數**,掃一遍維護最小值,O(N log N)。

```cpp
// sketch:依 X 升序掃,數「Y 是前綴嚴格最小值」的點
sort(p.begin(), p.end());            // 依 X 升序(X 是排列)
int mn = INT_MAX, ans = 0;
for (auto [x, y] : p)
    if (y < mn) { ans++; mn = y; }   // 前綴新低 ⇔ 矩形內部無點
```

### [D. Accomplice](https://atcoder.jp/contests/abc462/tasks/abc462_d)
`差分` `組合計數` `~1200`

::: info 題意
給定 N 名嫌犯(2≤N≤2×10^5)與整數 D(1≤D≤10^6);第 i 人於時刻 S_i 進館、T_i 離館(1≤S_i≤T_i≤10^6),期間不進出。

已知兇手恰兩人,犯行自某整數時刻 x 開始、於 x+D 完成,兩名共犯在 [x,x+D] 全程在館(恰於 x 進館或恰於 x+D 離館也算)。

求:(無序共犯二人組, 開始時刻 x) 的可能組合數。

例:`D=1`,三人在館 `[1,3], [2,4], [1,4]` → `x=1` 全程在館者 {1,3}、`x=2` 為 {1,2,3}、`x=3` 為 {2,3} → 答案 C(2,2)+C(3,2)+C(2,2) = 1+3+1 = 5。
:::

::: tip 作法
時刻 t 貢獻 C(c_t,2),c_t = 區間 [t,t+D] 全程在館人數。

**把每人退館時刻改成 T_i−D(滯留<D 者忽略),c_t 就退化成普通的「時刻 t 在館人數」**;差分/imos 掃出所有 c_t 累加,O(N+M)。

```cpp
// sketch:退館改記 T_i - D 後做 imos
for (int i = 0; i < n; ++i) {
    if (T[i] - S[i] < D) continue;       // 滯留 < D:不可能全程在館
    diff[S[i]]++, diff[T[i] - D + 1]--;  // 可行 x ∈ [S_i, T_i - D]
}
long long c = 0, ans = 0;
for (int t = 1; t <= M; ++t)
    c += diff[t], ans += c * (c - 1) / 2;  // Σ C(c_t, 2)
```

### [E. Alternating Costs](https://atcoder.jp/contests/abc462/tasks/abc462_e)
`數學` `分類討論` `~1600`

::: info 題意
每筆測資給整數 A,B,X,Y(1≤A,B≤10^9,|X|,|Y|≤10^9),共 T≤2×10^5 筆。

棋子自 (0,0) 出發,每步向上下左右移動一格;第 k 步若 k 為奇數,橫移費用 A、縱移費用 B,k 為偶數則相反(橫 B、縱 A)。

求:每筆輸出把棋子移到 (X,Y) 的最小總費用。

例:`A=1, B=10`,目標 `(0,2)` → 直走「上、上」花 10+1=11(第 1 步縱移費 B);繞「右、上、左、上」四步全花 A,共 4,即為最小費用。
:::

::: tip 作法
對稱性設 X,Y≥0;X+Y 奇數時退一步 min(f(X−1,Y)+A, f(X,Y−1)+B) 化為偶數。

偶數時共 2K 步、奇偶步各 K 次,**A,B 與 X,Y 都可 swap 成 A≤B、X≤Y,最小費用為一次函數 g(K)=2KA+(Y−K)(B−A)**;端點取極值,答案 = min(g((X+Y)/2), g(Y)),每筆 O(1)。
:::

### [F. More ABC](https://atcoder.jp/contests/abc462/tasks/abc462_f)
`dp` `字串` `~2400`

::: info 題意
每筆測資給大寫英文字串 S(3≤|S|≤3×10^5,全部測資 Σ|S|≤3×10^5)與整數 K(1≤K≤10)。

一次替換 = 把 S 的任一字元改成任意大寫字母;目標是讓 ABC 作為連續子字串的出現次數恰好比原字串多 K 個。

求:達成目標所需的最少替換字元數;不可能則輸出 −1。

例:`K=1`:`S="ABX"` 改 `X→C` 即多 1 個 ABC,答案 1;`S="ABC"` 長度 3 至多容納 1 個 ABC 而原本已有 1 個 → 答案 −1。
:::

::: tip 作法
dp[i][j] = 前綴 S'₁..ᵢ 比原串至少多 j 個 ABC 的最少改字數;**每個位置只分「S'₍ᵢ₋₂..ᵢ₎=ABC」成立與否兩種轉移**:dp[i][j]=min(dp[i−3][j−1+Xᵢ]+Yᵢ, dp[i−1][j+Zᵢ]),X/Y/Z 皆 O(1) 預處理,整體 O(NK)。

另解:f(x)=「多出 ≥x 個 ABC 的最小花費」為凸,Alien DP 二分免除額 O(N log N)。

```cpp
// sketch:DP 轉移(X/Y/Z 皆由原串 O(1) 預處理)
// dp[i][j] = 前綴 i、比原串至少多 j 個 ABC 的最少改字數
for (int i = 3; i <= n; ++i)
    for (int j = 0; j <= K; ++j)
        dp[i][j] = min(dp[i-3][j-1+X[i]] + Y[i],  // 令 S'[i-2..i]="ABC"
                       dp[i-1][j+Z[i]]);          // S'[i-2..i]≠"ABC"
```
:::

### [G. Completely Wrong](https://atcoder.jp/contests/abc462/tasks/abc462_g)
`容斥` `組合計數` `多項式` `NTT` `分治` `~2600`

::: info 題意
箱中有 N 顆球(N≤2×10^5),球 i 塗顏色 C_i;另給長度 N 的序列 G(1≤C_i,G_k≤N)。

依序做 N 次操作:第 k 次自箱中等機率抽出一球並丟棄,若其顏色恰為 G_k 得 1 分。

求:N 次操作後總得分為 0 的機率,以 mod 998244353 的有理數取模形式輸出。

例:`N=2`,`C=[1,2]`,`G=[2,1]` → 抽序 (球1,球2) 顏色依序 (1,2) 全程不中;抽序 (球2,球1) 第一抽顏色 2 恰為 G_1 → 得分 0 的機率 = 1/2,輸出 499122177。
:::

::: tip 作法
抽球順序即隨機排列 P,總分 0 ⟺ 全程 C_{P_k}≠G_k,容斥成 (1/N!)Σ_A (−1)^|A| |∩_{i∈A} Sᵢ|;

記 C、G 中值 k 的個數為 X_k,Y_k,固定各 k 被配對的次數後**貢獻按顏色獨立,化成 f_k(x)=Σ_j (−1)ʲ C(X_k,j) C(Y_k,j) j! xʲ,取 f=∏ f_k 則答案 = (1/N!)Σ cᵢ(N−i)!,且 ∏ 總次數 ≤N**;

小次數優先合併的分治 NTT,O(N log²N)。
:::
