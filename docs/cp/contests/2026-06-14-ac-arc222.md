### [A. Colorful Intervals](https://atcoder.jp/contests/arc222/tasks/arc222_a)
`構造` `~900`

::: info 題意
給定正整數 N、M 與 M 個區間 (L_j, R_j)(1≤L_j≤R_j≤N;多測,ΣN、ΣM ≤ 2×10^5)。

構造一個長度 N 的正整數序列 A,使每個區間 [L_j, R_j] 內的元素兩兩相異(合法序列保證存在)。

輸出:任一個使 max(A_1,…,A_N) 最小的合法序列;多解輸出任意一組。

例:`N=4`,區間 `[1,3]`、`[2,4]` → 最長區間長 d=3,構造 `A = (1,2,3,1)`:區間 [1,3] 取值 {1,2,3}、[2,4] 取值 {2,3,1} 皆兩兩相異;又 [1,3] 內 3 格須相異逼出 max ≥ 3,故 3 已最優。
:::

::: tip 作法
答案 = 最長區間長度 `d`:下界因該區間 `d` 個位置須兩兩相異;上界直接構造 `A_i = ((i-1) mod d) + 1`,任何長 ≤ `d` 的區間 mod d 後不重複。
:::

### [C. 2 Directions vs 4 Directions](https://atcoder.jp/contests/arc222/tasks/arc222_c)
`博弈` `dp` `~2300`

::: info 題意
給定 N×N 棋盤,初始全黑,以及每格由黑塗白的費用矩陣 A_{i,j}(2≤N≤500,多測 ΣN²≤500²,0≤A_{i,j}≤10^9)。

裁判指定起點後,Alice 先付費塗白任意若干格;接著棋子放上起點,從 Alice 開始輪流移動:Alice 只能左右移一格,Bob 可上下左右移一格(不得出界),兩人各走 10^10 步後,棋子停在白格則 Alice 勝,黑格則 Bob 勝;雙方皆最優。

輸出:對每個起點 (h,w),求 Alice 保證獲勝所須付的最小塗白總費用(共 N×N 個答案)。
:::

**關鍵**:Bob 能永久守住任一曾到達的格,故 type-A 的左右鄰格不白就被反殺;充分性靠 Alice 永遠移回 type-A。

每個起點上下兩段獨立,對行做簡單 DP 求最小塗白成本。

```cpp
// sketch:對行 DP(向下方向;向上 up[][] 對稱)
// cost(r,c) = (r,c) 左右鄰格(type-B)的塗白費用和;出界的 c±1 視為 INF
dn[N][c] = cost(N, c);
for (int r = N - 1; r >= 1; --r)
    for (int c = 1; c <= N; ++c)
        dn[r][c] = cost(r, c) + min(dn[r+1][c-1], dn[r+1][c+1]);  // 列差 ±1
// 起點 (h,w):上段 + 下段獨立 → ans = up[h][w] + dn[h][w] - cost(h,w)
```

### [D. Shift and Add](https://atcoder.jp/contests/arc222/tasks/arc222_d)
`數位dp` `~3200`

::: info 題意
給定兩個長度 N 的正整數序列 A、B(N≤2×10^4,1≤A_i,B_i≤10^9−1,即加數可達 9 位)。

定義 X_0=0,每步 i 二選一:X_i = 10X_{i-1}+A_i 或 X_i = 10X_{i-1}+B_i。

輸出:對每個 k=1,…,N **獨立**求 X_k 的十進位桁和(數字和)最小值;不同 k 可採不同的選擇方式。

例:`A = [99, 10]`、`B = [11, 1]` → k=1 最佳選 B_1:X_1=11,桁和 2;k=2 最佳改選 A_1、A_2:X_2 = 10·99+10 = 1000,桁和 1(進位把 990 清成 1000)——不同 k 的最佳選法不同。
:::

::: tip 作法
`X_i` 低 9 位只由最近 9 次選擇決定(≤2^9 種),高位幾乎是整串平移。

**寶石**:高位只可能因低位將來進位 `+1`,故狀態 `dp[最近9次選擇][x∈{0,1}]` 同時記「高位桁和最小值」與「高位 +1 後桁和最小值」;要對所有 k 輸出,把低位→高位 DP 視為 DAG 最短路,轉置成高位→低位 DP。
:::

### [E. XOR Matching](https://atcoder.jp/contests/arc222/tasks/arc222_e)
`XOR` `WHT` `根號` `~2800`

::: info 題意
給定 N 張卡片(2≤N≤2×10^5),第 i 張寫有整數 A_i∈[0, 2^M),M≤20。

對每個 X∈[0, 2^M),定義 f(X) 為「每對由兩張相異卡片組成、兩卡 XOR 恰為 X、每張卡至多屬於一對」時可配成的最大對數。

輸出:單一整數 Σ_{X=0}^{2^M−1} f(X)·10^X mod 998244353。

例:M=1、卡片 `(0, 0, 1)` → f(0)=1(兩張 0 互配),f(1)=1(一張 0 配上 1,另一張 0 落單)→ 答案 1·10^0 + 1·10^1 = 11。
:::

::: tip 作法
設值頻率 `C`,則 X≠0 時 `f(X) = ½·Σ_{i⊕j=X} min(C[i],C[j])`(X=0 為 Σ⌊C/2⌋)。

**關鍵**:`min(a,b) = Σ_x [a≥x][b≥x]`,對每個閾值 `x≤K` 把指示陣列與自身做 XOR 卷積(WHT);`min>K` 的對數少,直接暴力枚舉修正。取 `K=(N²/M2^M)^{1/3}` 得 `O((NM2^M)^{2/3})`。

```cpp
// sketch:min 拆閾值 —— 小閾值走 WHT,高頻率暴力補
for (int x = 1; x <= K; ++x) {
    for (int v = 0; v < (1 << M); ++v) g[v] = (C[v] >= x);  // 指示陣列
    wht(g);                                  // Walsh–Hadamard 變換
    for (auto &t : g) t = t * t;             // 與自身做 XOR 卷積
    iwht(g);                                 // g[X] = #{(i,j): i^j=X 且 C_i,C_j ≥ x}
    for (int X = 1; X < (1 << M); ++X) sum2[X] += g[X];  // 累出 2·f(X)
}
// min(C_i,C_j) > K 的部分:頻率 > K 的值僅 O(N/K) 個,兩兩暴力補足
```
:::

### [F. Triple Transformation](https://atcoder.jp/contests/arc222/tasks/arc222_f)
`不變量` `BSGS` `數論` `~4000`

::: info 題意
給定非負整數三元組 A=(A_1,A_2,A_3) 與 B=(B_1,B_2,B_3)(值 ≤10^8;多測 T≤300)。

一次操作把 (x,y,z) 依規則替換:若某元素嚴格大於另兩者之和(如 y+z<x)則變 (x−y−z, 2y, 2z)(另兩種情況對稱);否則變 (y+z−x, x+z−y, x+y−z);每個三元組恰落在四種情況之一。

輸出:將 A 操作成 B 的最少操作次數;不可達則輸出 -1。

例:A=(3,1,1)、B=(1,2,2) → 3>1+1,一步變 (3−2, 2·1, 2·1)=(1,2,2),答案 1;對照不變量:S=5 不變,且 (1,2,2) ≡ 2·(3,1,1) (mod 5)。
:::

::: tip 作法
**寶石**:`S=x+y+z` 不變,且 mod S 下每步整組只變成 `(2x,2y,2z)` 或 `(-2x,-2y,-2z)`。先用約 `log S` 次操作化為 S 為奇數的情形,問題化簡為解 `2^K·A_i ≡ b_i (mod S)` 的最小 K → **離散對數變種**,BSGS 拆 `K=iL+j`,`O(√S log S)`。
:::
