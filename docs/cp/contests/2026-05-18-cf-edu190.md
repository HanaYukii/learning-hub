### [A. Optimal Purchase](https://codeforces.com/contest/2230/problem/A)
`貪心` `數學` `~800`

::: info 題意
給定 $n$ 個學生。商店賣兩種鑰匙:單人鑰匙每把 $a$ 元供 $1$ 人;團體鑰匙每把 $b$ 元供**至多** $3$ 人(少用不退差價)。

**求**讓全部 $n$ 人都拿到權限的最小花費。(多測)

例:$n=4,a=2,b=3$ → 一把團體(3 人)$+$ 一把單人 $=5$;全單人 $8$、兩把團體 $6$,故答案 $5$。
:::

::: tip 作法
最優解只有三種形態——全單人 $n\cdot a$、$\lfloor n/3\rfloor$ 把團體 $+$ 餘數單人、$\lceil n/3\rceil$ 把團體——**取 $\min$ 即可**。
:::

### [B. Digit String](https://codeforces.com/contest/2230/problem/B)
`字串` `貪心` `~1100`

::: info 題意
給定僅含數字 $1\sim 4$ 的字串 $s$。稱 $s$ 美麗 ⇔ 無法選出任何子序列(依原順序寫出)組成 $4$ 的倍數;空字串視為美麗。

**求最少刪除幾個字元**使 $s$ 美麗。(多測)

例:`1322` → 子序列 `12` 是 4 的倍數;刪兩個 `2` 留 `13`(子序列僅 1、3、13,皆非 4 倍數)→ 答案 $2$;只刪一個救不了(`322`/`122`/`132` 分別含 `32`/`12`/`32`)。
:::

::: tip 作法
**除 4 只看末兩位** → 用 1–4 拼得出的危險子序列只有 `4` 與「(1∣3) 之後出現 2」;美麗 ⇔ 刪光 `4` 且保留串形如 `2…2` 接 `(1∣3)…` → 枚舉分割點最大化「前段 2 數 $+$ 後段 1/3 數」,$O(n)$。
:::

### [C. Arrange the Numbers in a Circle](https://codeforces.com/contest/2230/problem/C)
`構造` `貪心` `分類討論` `~1500`

::: info 題意
手上有 $c_i$ 張寫著 $i$ 的卡($i=1,\dots,n$)。要選**至少 3 張**排成一圈,使每三張環狀連續的卡 $a_i,a_{(i+1)\bmod k},a_{(i+2)\bmod k}$ 中至少兩張相等。

**求**最多能排上幾張卡;排不出合法圈則輸出 $0$。(多測)

例:$c=(1,1,4)$ → 圈 `3 3 1 3 3 2`:六個長度-3 窗口(含跨接縫)都含兩張 3 → 全部 $6$ 張可用。

### [D. Good Schedule](https://codeforces.com/contest/2230/problem/D)
`dp` `分類討論` `~1800`

::: info 題意
Alice、Bob 追同一部 $n$ 集的劇:第 $i$ 天 Alice 城播第 $a_i$ 集、Bob 城播第 $b_i$ 集。選定區間 $[L,R]$ 後兩人從零開始:當天播的恰是自己的「下一集」(前面全看過、這集沒看)才會看,否則整天沒看。

**求**有幾個區間 $[L,R]$,使區間內**每一天**兩人都「同看同一集」或「同沒看」。(合法區間數可達 $\Theta(n^2)$,答案開 64-bit。)

例:$a=[1,2,1]$、$b=[1,2,2]$ → 合法 $[1,1],[1,2],[1,3],[2,2]$ 共 $4$($[3,3]$:Alice 獨看第 1 集破功;$[2,3]$ 同理)。
:::

::: tip 作法
固定 $L$ 時合法 $R$ 是一段前綴 → 只要算「破功日」。**破功與否只看下一集在兩序列後綴的首現位置是否相同**:相同 → 兩人同天看它,跳到那天繼續等再下一集;不同 → 較早出現處有人獨看,當天破功。倒掃把後綴 dp 串成鏈,$O(n)$。

```cpp
// sketch:倒掃 + 後綴 dp(集數 0-indexed;pa/pb/dp 全初始化為 n)
for (int i = n - 1; i >= 0; --i) {
    pa[a[i]] = i; pb[b[i]] = i;               // 集數 x 在後綴 [i..] 的首現
    if (a[i] == b[i]) {                        // 狀態:兩人剛同步看完 a[i]
        int x = a[i] + 1;                      // 一起等下一集
        dp[i] = (pa[x] == pb[x]) ? dp[pa[x]] : min(pa[x], pb[x]);
    }
    int brk = (pa[0] != pb[0]) ? min(pa[0], pb[0]) : dp[pa[0]];
    ans += brk - i;                            // L=i 時 R ∈ [i, brk-1]
}
```
:::

### [E. Minimum Influence](https://codeforces.com/contest/2230/problem/E)
`分類討論` `貪心` `~2100`

::: info 題意
$n$ 則新聞各有政治度 $p_i$、文化度 $c_i$;$m$ 個使用者各有容忍度 $tp_j,tc_j$ 與影響區 $d_j$(值域皆 $\le 10^6$)。單邊影響:值 $<$ 容忍 → $0$;落在 $[\text{容忍},\ \text{容忍}+d)$ → 取原值;$\ge$ 容忍 $+d$ → 封頂為容忍 $+d$。總影響 $I(i,j)$ 為政治、文化兩邊之和。

對每個使用者 $j$,**輸出** $\min_i I(i,j)$。

例:使用者 $(tp,tc,d)=(10,10,20)$:新聞 $(100,12)$ → 政治封頂 $30$、文化取原值 $12$,$I=42$;新聞 $(100,100)$ → $30+30=60$;答案 $42$。
:::

::: tip 作法
最優新聞五選一:**$I(i,j)\le p_i+c_i$ 恆成立 → 「兩邊都落在中段」的情形被全域 $p_i+c_i$ 最小的那則統包**;其餘 case 是某一邊貼 $0$(值 $<$ 容忍)或貼封頂(值 $\ge$ 容忍 $+d$)——**$I$ 對單邊值單調不減 → 每個 case 只需該範圍內「另一邊的最小值」**,對值域開前綴/後綴 min 即 $O(1)$ 查,總 $O(n+m+A)$。

```cpp
// sketch:I(val,l,r) = (val < l ? 0 : min(val, r));五個候選取 min
long long best = I(p[mid],tp,tp+d) + I(c[mid],tc,tc+d);  // mid = 全域 p+c 最小
for (int k = 0; k < 2; ++k) {              // 第 k 邊貼 0 / 貼封頂
    int lo = tol[k], hi = min(A - 1, tol[k] + d);
    if (pre[k][lo] < INF)                  // 該邊值 < 容忍:貢獻 0
        best = min(best, I(pre[k][lo], tol[k^1], tol[k^1] + d));
    if (suf[k][hi] < INF)                  // 該邊值 ≥ 容忍+d:貢獻 hi
        best = min(best, hi + I(suf[k][hi], tol[k^1], tol[k^1] + d));
}
```
:::

### [F. Game on Growing Tree](https://codeforces.com/contest/2230/problem/F)
`博弈` `樹` `dp` `分治` `~2600`

::: info 題意
遊戲:樹上所有點初始白色。Alice 先手:第一手選任一點放棋子並染紅;之後每手須把棋子移到**相鄰白點**並染紅,無白鄰點則遊戲結束。Bob 每手任選一個白點染藍(永久封鎖)。得分 $=$ 紅點數,Alice 最大化、Bob 最小化,雙方最優。

給定從單點 $1$ 長出的樹:$q$ 次加點($q\le 2\cdot 10^5$),第 $i$ 次把新點 $i+1$ 掛到 $v_i$。**每次加點後輸出**當前樹的遊戲值。

例:三次全掛在點 $1$ → 輸出 `1 2 2`:兩點時 Bob 直接封死唯一鄰點;星形時 Alice 從中心起步,Bob 每回合只封得掉一片葉 → 恰拿 $2$ 個紅點,再多拿不到。
:::

::: tip 作法
單棵樹:Alice 每到一點,Bob 恰有一手可先動 → **封死最佳分支入口最優 ⇒ $f(v)=1+$ 次大(相鄰分支值)**;用 top-3 down 值 $+$ 換根 up 值處理「排除來向」,$O(n)$ 對全點取最大即遊戲值。

線上部分:$f=1+$次大 ⇒ 值 $k$ 需要兩個值 $k{-}1$ 的分支 → **答案 $\le\log_2 n+1\approx 20$;且加點只擴充 Alice 的選項 → 答案隨時間單調不減 → 對 query 軸做答案值域分治**,整棵重算僅 $O(20\cdot\log q)\approx 350$ 次、每次 $O(n)$。

```cpp
// sketch:值域分治 —— ans[l..r) 單調不減且落在 [L,R]
int game_value(int m);                    // 第 m 個 query(0-indexed)後整棵重算:O(size)
void rec(int l, int r, int L, int R) {
    if (l >= r) return;
    if (L == R) { fill(ans + l, ans + r, L); return; }
    int mid = (l + r) / 2;
    ans[mid] = game_value(mid);
    rec(l, mid, L, ans[mid]); rec(mid + 1, r, ans[mid], R);
}
```
:::
