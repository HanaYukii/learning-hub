# Codeforces Round 1104 (Order Capital Round 2, Div. 1 + Div. 2) — 學習重點

> 難度 `~` 為 CF rating(估計)。

### [A. Destroying Towers](https://codeforces.com/contest/2237/problem/A)
`貪心` `~800`

::: info 題意
給定 n≤100 座塔排成一列的高度陣列 a(正整數)。

每座塔恰好被操作一次、順序自選:站上塔 i 向右發射雷射,把右邊第一座「當前高度」嚴格更高的塔削成與塔 i 同高(不存在這種塔則什麼都不發生)。

求/輸出:所有操作順序中,最終高度總和的最小值。

例:`a = [3, 1, 2]` → 先操作塔 2(高 1):右邊第一座更高的是塔 3,削成 1;塔 1 最左無人能削、塔 2(1)不比塔 1(3)高也不會被削 → 最終 `[3, 1, 1]`,總和 5 = 3+1+1(前綴最小和)。
:::

::: tip 作法
答案 = Σᵢ min(a₁..aᵢ)——**每座塔最終恰能被削到其前綴最小值**(適當順序可達成,且不可能更低)。
:::

### [B. Annoying the Ghost](https://codeforces.com/contest/2237/problem/B)
`貪心` `逆序對` `~1200`

::: info 題意
給定長 n≤2000 的鴨堆數量陣列 a,以及嚴格遞增的目標序列 b。

過程分兩階段:第一階段可對每堆各加任意非負整數;第二階段只能反覆交換相鄰兩堆;兩階段結束後序列必須恰好等於 b。

求/輸出:所有合法過程中第二階段的最少交換次數;不存在合法過程則輸出 -1。

例:`a = [2, 1]`,`b = [1, 3]` → `a₁=2` 配到 3、`a₂=1` 配到 1,目標序列 `[3, 1]` 逆序對 = 1;對應過程:第一階段加成 `[3, 1]`,一次相鄰交換得 `[1, 3]` → 答案 1。
:::

::: tip 作法
每個 `a[i]` **配到不小於它的最小可用 `b[j]`**(`lower_bound` 後刪除),配不到則 -1;所得目標序列的逆序對數即答案。

貪心正確性:小的 b 留給大的 a 反而會卡死。

```cpp
// sketch:貪心配對 + 逆序對
multiset<int> pool(b.begin(), b.end());
vector<int> t(n);
for (int i = 0; i < n; ++i) {
    auto it = pool.lower_bound(a[i]);   // 不小於 a[i] 的最小可用 b
    if (it == pool.end()) return -1;    // 配不到
    t[i] = *it; pool.erase(it);
}
// 答案 = t 的逆序對數(BIT / 歸併排序)
```
:::

### [C. Duck Surplus](https://codeforces.com/contest/2237/problem/C)
`貪心` `構造` `~1400`

::: info 題意
給定長 n≤2e5 的鴨堆數量陣列 a(a_i≤1e9)。

只要序列還不是不減的,就必須選一個相鄰逆序對 a_i>a_{i+1},把這對換成 (a_{i+1}, a_i+a_{i+1})(交換後右邊變成兩者之和);每步選哪個位置自由,且保證無論怎麼選過程都會終止於不減序列。

求/輸出:過程結束時最大堆的最小可能值。

例:`a = [3, 1, 2]` → 唯一逆序對 `(3,1)` 換成 `(1,4)` 得 `[1, 4, 2]`;再處理 `(4,2)` 得 `[1, 2, 6]`:3 吞了 1 再吞 2,答案 6(一遍掃:ans 依序 3→4→6)。
:::

::: tip 作法
一遍掃:`if ans > a[i]: ans += a[i] else ans = a[i]`,輸出最終 ans。

**寶石:固定做最左可行操作後每個數命運唯一——它會一路向右「吞掉」比它小的後續元素**,答案即最後一任霸主的吞併和。
:::

### [D. Fullmetal Bitchemist](https://codeforces.com/contest/2237/problem/D)
`不變量` `組合計數` `~1700`

::: info 題意
給定長 n≤1e6 的 01 字串 s。

定義字串「美麗」:可反覆執行「選兩個相等的相鄰字元,刪掉並在原處插入一個相反字元」(即 00→1、11→0),最終縮到長度恰為 1(如 10001 美麗;111 只能變成 01 後卡死,不美麗)。

求/輸出:s 的非空美麗子字串(連續)個數。

例:`s = "101"` → 6 個子串中差 ≢0 (mod 3) 的有 `1`、`0`、`1`、`101` 共 4 個;但 `101` 是長 3 的交替串、無相同相鄰對可動 → 扣掉 1,答案 3(三個單字元子串)。
:::

::: tip 作法
操作(00→1、11→0)都使 #1−#0 變 ±3,故 **`(#1 − #0) mod 3` 是不變量**,美麗 ⟺ 差 ≢0 (mod 3);唯一例外是**長 ≥3 的奇長交替串(差 ±1 卻無相同相鄰對可動)**——按前綴殘差配對計數後,對每個結尾扣 `(y-1)//2`(`y` 為結尾交替段長)。

```cpp
// sketch:前綴殘差計數 + 交替段修正
long long ans = 0; int cnt[3] = {1, 0, 0};  // 前綴差 mod 3(含空前綴)
int d = 0, y = 1;                           // y:以 i 結尾的交替段長
for (int i = 0; i < n; ++i) {
    d = ((d + (s[i] == '1' ? 1 : -1)) % 3 + 3) % 3;
    ans += i + 1 - cnt[d];                  // 差 ≢0 (mod 3) 的結尾子串
    if (i && s[i] != s[i-1]) ++y; else y = 1;
    ans -= (y - 1) / 2;                     // 扣長 ≥3 的奇長交替串
    ++cnt[d];
}
```
:::

### [E. Permutation Commutation](https://codeforces.com/contest/2237/problem/E)
`排列` `置換環` `構造` `~2000`

::: info 題意
給定長 n≤2e5 的排列 a,以及缺項序列 b:每個元素是 -1 或 1..n 中的整數,且已填的值互不相同。

要把每個 -1 補上,使 b 成為與 a 可交換的排列,即對所有 i 滿足 a_{b_i}=b_{a_i}。

求/輸出:字典序最小的合法補完 b;不存在則報告無解。

例:`a = [2,1,4,3]`(A 的環:`(1 2)(3 4)`),`b = [3,-1,-1,-1]` → 由 b_{a_1}=a_{b_1} 傳播得 b₂=a₃=4;剩下位置環 `(3 4)` 配未用值環 `(1 2)` 的最小起點 → `b = [3,4,1,2]`,四個 i 皆滿足 a_{b_i}=b_{a_i}。
:::

::: tip 作法
**可交換 ⟺ B 把 A 的每個 cycle 映到同長度的 cycle**;沿 A 的環用 b_{a_i}=a_{b_i} 傳播已知值並檢查一致/重複,剩餘未定環按長度分組,位置最小者由小到大依序配上等長「未用值環」的最小起點——即為字典序最小。
:::

::: tip 作法
合法 ⟺ `a_1=1, a_n=m`,且每相鄰對滿足 `a_{i+1}=a_i+1` 或 `a_i=m` 或 `a_{i+1}=1`;

**`dp[i][j]` = 前綴以值 j 結尾時最多保留幾個位置不改**,轉移只有 shift 與前綴/後綴 max,壓到 O(n+m),答案 `n − dp[n][m]`。

```cpp
// sketch:O(nm) 概念版(實作用整體平移 + 前綴/後綴 max 壓到 O(n+m))
dp[1] = (a[1] == 1);                 // 其餘 = -INF;邊界:a_1 必須是 1
for (int i = 2; i <= n; ++i) {
    int best = *max_element(dp + 1, dp + m + 1);  // 供 j=1(重開新段)
    int pm = dp[m];                               // 前一格是 m,接什麼都行
    for (int j = m; j >= 1; --j)                  // 倒序原地 shift
        dp[j] = max({ j > 1 ? dp[j-1] : -INF,     // a_i = a_{i-1} + 1
                      pm,
                      j == 1 ? best : -INF }) + (a[i] == j);
}
// 答案 = n - dp[m](a_n 必須是 m)
```

### [G. Send GCDs](https://codeforces.com/contest/2237/problem/G)
`互動` `數論` `構造` `~2700`

::: info 題意
run-twice 通訊互動題,同一份程式跑兩次、兩輪間不能直接傳資訊。

第一輪(Ja)讀入 n≤1e3 與陣列 a(a_i≤1e6),須送出長 k≤⌈10n/9⌉+150 的正整數陣列 b(b_i≤1e6)給裁判;第二輪(Quack)只拿到 n 與 k,可向裁判詢問至多 180n+150 次 gcd(b_i,b_j)(i≠j)。

求/輸出:第二輪據詢問結果正確還原並輸出整個陣列 a。

例:探針含 2¹⁹=524288、3¹²=531441、5⁸=390625;601-smooth 的 720=2⁴·3²·5 與這三支取 gcd 得 16、9、5(其餘探針 gcd=1),相乘 = 720 → 自身被精確還原。

::: tip 作法
b 開頭放前 110 個質數(至 601)在 1e6 內的最大冪當「探針」——任一 601-smooth 數對全部探針取 gcd 再相乘即還原自身,故 2¹⁸ 個 smooth number 可精確傳輸。

**寶石:進位制重編碼——每 9 個 20-bit 的 a 值重切成 10 個 18-bit 塊**(`(10⁶)⁹ < (2¹⁸)¹⁰`),陣列長壓到 ~10n/9。
:::

::: tip 作法
把每次進食視為對「上一目標→新目標」的路徑染上新時間戳:樹剖拆段,每條重鏈用 `map`(ODT 式區間賦值)維護時間戳區段,染色時逐段切割並統計新覆蓋量累加進答案。

```cpp
// sketch:單條重鏈上的 ODT —— map<段起點, 時間戳>,區間賦值
auto split = [&](int x) {                    // 保證 x 成為某段起點
    auto it = prev(odt.upper_bound(x));
    odt.emplace(x, it->second);
};
split(l); split(r + 1);                      // 切出 [l, r]
for (auto it = odt.find(l); it != odt.end() && it->first <= r; ) {
    // 逐段:依此段舊時間戳統計新覆蓋量,累加進答案
    it = odt.erase(it);
}
odt[l] = t;                                  // 整段換上新時間戳
```
