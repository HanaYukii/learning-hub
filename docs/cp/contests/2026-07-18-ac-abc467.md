### [D. Concentric Circles](https://atcoder.jp/contests/abc467/tasks/abc467_d)
`幾何` `AtC~950`

::: info 題意
給定四個點 $P,Q,R,S$(座標為整數、絕對值 $\le 10^9$;保證 $P\ne Q$、$R\ne S$)。問是否存在兩個**同心**圓 $C_1,C_2$:$P,Q$ 都在 $C_1$ 上、$R,S$ 都在 $C_2$ 上(兩圓可重合)。$T\le 5\times10^4$ 筆測資。

例:$P=(1,0)$、$Q=(-1,0)$、$R=(0,2)$、$S=(0,-2)$ → 圓心取原點,半徑 $1$ 與 $2$ → Yes。$P=(0,0)$、$Q=(2,0)$、$R=(1,0)$、$S=(3,0)$ → 兩條中垂線是 $x=1$ 與 $x=2$,平行不重合 → No。
:::

::: tip 作法
先把「同心圓存在」翻成幾何條件:圓心 $O$ 要同時滿足 $|OP|=|OQ|$ 與 $|OR|=|OS|$,而**到兩點等距 ⟺ 落在兩點的中垂線上**。所以整題就是問:**PQ 的中垂線和 RS 的中垂線有沒有交點**——不平行必有交點 → Yes;平行時要是同一條線才 Yes。

中垂線由 $|X-P|^2=|X-Q|^2$ 移項,得整數係數直線 $ax+by=c$:$a=2(Q_x-P_x)$、$b=2(Q_y-P_y)$、$c=|Q|^2-|P|^2$。平行與否看兩條線法向量的叉積 $a_1b_2-a_2b_1$。

**坑:座標 $10^9$ 時 long long 會爆**——$a,b$ 到 $4\times10^9$,叉積到 $1.6\times10^{19}$,超過 long long 上限 $9.2\times10^{18}$;共線判定 $a_1c_2-a_2c_1$ 更到 $8\times10^{27}$。**全程 `__int128`** 最省事。

```cpp
using lll = __int128;
lll a1 = 2*(qx - px), b1 = 2*(qy - py);
lll c1 = (lll)qx*qx + (lll)qy*qy - (lll)px*px - (lll)py*py;
lll a2 = 2*(sx - rx), b2 = 2*(sy - ry);
lll c2 = (lll)sx*sx + (lll)sy*sy - (lll)rx*rx - (lll)ry*ry;
bool yes = (a1*b2 - a2*b1 != 0)                        // 不平行 → 必有交點
        || (a1*c2 - a2*c1 == 0 && b1*c2 - b2*c1 == 0); // 平行:同一條線才行
```

### [E. Adjacent Sums (hard)](https://atcoder.jp/contests/abc467/tasks/abc467_e)
`數學` `構造` `AtC~1700`

::: info 題意
給定序列 $A$(長 $N$)與 $B$(長 $N-1$),元素皆在 $0..M-1$($2\le N\le 2\times10^5$、$3\le M\le 10^9$)。可對任意 $A_i$ **+1 任意次**。

**求/輸出**:使每個相鄰對滿足 $(A_i+A_{i+1})\bmod M=B_i$ 的最少 +1 總次數。(C 題是 $M=2$ 的簡化版。)

例:$M=5$、$A=[0,0]$、$B=[3]$ → 給 $A_2$ 加 $3$ 次($(0+3)\bmod 5=3$),且 $c_1+c_2\equiv 3$ 下 $c_1+c_2$ 最小就是 $3$ → 答案 $3$。
:::

::: tip 作法
**第一步:把問題翻成同餘條件。** 設位置 $i$ 總共加了 $c_i$ 次,令 $d_i=(B_i-A_i-A_{i+1})\bmod M$,那麼每個相鄰對的條件就是 $(c_i+c_{i+1})\equiv d_i \pmod M$。關鍵在:一旦**固定 $c_1=x$,整條鏈就被鎖死**——$c_2\equiv d_1-x$、$c_3\equiv d_2-c_2$、依此類推,每個 $c_i$ 都是 $\pm x$ 加上一個常數(偶數位是 $+x$、奇數位是 $-x$)。而每個位置只需加「餘數那麼多」次(多加整圈 $M$ 純浪費),所以 $\mathrm{cost}(x)=\sum_i\bigl((\pm x + c_i^{(0)})\bmod M\bigr)$。

暴力做法是把 $x\in[0,M)$ 全試一遍取 $\min$——但 $M$ 可到 $10^9$,$O(NM)$ 直接 TLE。所以要看清楚 $\mathrm{cost}(x)$ 這個函數長什麼樣。

**第二步:$\mathrm{cost}(x)$ 是鋸齒和。** 每一項 $(\pm x+c)\bmod M$ 對 $x$ 都是鋸齒:$+x$ 的位置每步 $+1$,直到 $x=M-c$ 時繞過一圈**瞬間掉 $M$**;$-x$ 的位置每步 $-1$,在 $x=c+1$ 時**瞬間升 $M$**。把 $N$ 個鋸齒加總:

固定斜率 $=$ 偶位數 $-$ 奇位數 $\in\{0,1\}$(非負!),其餘全是離散的 $\pm M$ 跳。**斜率非負代表兩個跳點之間函數只會緩升——谷底不可能出現在段中間,只可能在 $x=0$ 或某個 $-M$ 掉落點正後方。** 所以不必掃 $M$ 個點:收集每個位置的跳點($O(N)$ 個),排序後只在這些候選點求值,$\mathrm{cost}=\mathrm{cost}(0)+\text{斜率}\cdot x+\text{累積跳量}$,整體 $O(N\log N)$。順帶一提,鋸齒和**不是單峰函數,不能三分**——這是想抄捷徑會踩的坑。

**實作最容易錯的地方:同一個 $x$ 上可能同時疊著多個 $\pm M$ 的跳(有的掉、有的升),必須全部加完再取 $\min$**——只跳一半就比較,會抓到一個實際上不存在的假值(sample 3 正是這樣掛的)。

```cpp
// c[0]=0 連鎖出 c[i];cost(x):偶位 +x、奇位 −x(mod M)
ll cost0 = 0; for (int i = 0; i < n; i++) cost0 += c[i];
ll slope = (n + 1) / 2 - n / 2;                    // 偶位數 − 奇位數 ∈ {0,1}
vector<pair<ll,ll>> ev;                            // {跳點 x, ±M}
for (int i = 0; i < n; i++)
    if (i % 2 == 0) { if (c[i] >= 1)     ev.push_back({M - c[i], -M}); }
    else            { if (c[i] <= M - 2) ev.push_back({c[i] + 1,  M}); }
sort(ev.begin(), ev.end());
ll best = cost0, acc = 0;
for (size_t i = 0; i < ev.size();) {
    ll pos = ev[i].first;
    while (i < ev.size() && ev[i].first == pos) acc += ev[i++].second;  // ★同位置全加完
    best = min(best, cost0 + slope * pos + acc);
}
```

### [F. Email Scheduling Optimization](https://atcoder.jp/contests/abc467/tasks/abc467_f)
`貪心` `線段樹` `AtC~2050`

::: info 題意
給定正整數序列 $A,B$(各長 $N$)。寫 $N$ 封信,一次只能寫一封、順序自選:寫第 $j$ 封花 $A_j$ 分鐘,寫完瞬間寄出、$B_j$ 分鐘後收到回信。

$Q$ 次查詢:`1 i x`(改 $A_i=x$)或 `2 i x`(改 $B_i=x$),每次改完**輸出**最佳順序下「收齊所有回信」的最早時間($N,Q\le 10^5$、$A_j,B_j\le 10^9$)。

例:$A=[3,1]$、$B=[1,100]$ → 先寫 $2$:完成 $1$、回信 $101$;再寫 $1$:完成 $4$、回信 $5$ → 答案 $101$(反過來是 $104$)。
:::

::: tip 作法
**先解決「單次詢問怎麼排」。** 第 $j$ 封信的回信時間 $=$ 它寫完的時刻(排在它前面所有信的 $A$ 前綴和)$+ B_j$,答案是全部的 $\max$。直覺上回信等最久($B$ 最大)的要先寫——嚴格化就是標準**交換論證**:相鄰兩封信對調,只影響這兩封的完成時刻,比較兩種順序的 $\max$ 可知「$B$ 大的排前面」永遠不劣,而且這結論**跟 $A$ 完全無關**。

排序固定後,把答案改寫成**跟順序無關的封閉式**:對每封信 $j$,排在它前面(含自己)的恰是所有 $B_k\ge B_j$ 的信,故

$$\text{答案}=\max_j\Bigl(B_j+\sum_{B_k\ge B_j}A_k\Bigr).$$

這一步很關鍵——有了這個式子,「重排」就變成「維護數線上的統計量」,帶修改才有下手處。

**再解決「$Q$ 次修改怎麼維護」。** 把每封信放到 $B$ 值的數線上(離線收集所有出現過的 $B$ 值做座標壓縮),線段樹每個節點管一段 $B$ 值區間,存兩個量:`total`(區間內所有信的 $A$ 總和)與 `best`(區間內的 $\max_j(B_j+\text{區間內}\ B\ge B_j\ \text{的}\ A\ \text{和})$)。合併時左子(B 小)的每個候選都還要再加上右子(B 大)全部的 $A$——因為那些信都排在它前面:

`best = max(R.best, L.best + R.total)`

修改都是點更新:改 $A_i$ 就在 $B_i$ 那格加差值;改 $B_i$ 則把 $A_i$ 從舊格搬到新格(兩次點更新)。根的 `best` 即答案,總複雜度 $O((N+Q)\log N)$。

```cpp
void pull(int k) { total[k] = total[2*k] + total[2*k+1];
                   best[k]  = max(best[2*k+1], best[2*k] + total[2*k+1]); }
void add(int pos, ll d) {              // 在 B 值 = val[pos] 的格子加 d 的 A 量
    int k = sz + pos; total[k] += d;
    best[k] = (total[k] > 0) ? val[pos] + total[k] : NEG;
    for (k >>= 1; k >= 1; k >>= 1) pull(k);
}
// 答案 = best[1]
```
:::

### [G. Many Sweets Problem](https://atcoder.jp/contests/abc467/tasks/abc467_g)
`線段樹` `BIT` `資料結構` `AtC~2550`

::: info 題意
給定正整數陣列 $A$(長 $N$)。$Q$ 次查詢 `c x l r k`:先改 $A_c=x$,再回答——區間 $[l,r]$ 的糖果(甜度 $=A$ 值)中,吃到總甜度 $\ge k$ 的**最少顆數**;湊不到輸出 $-1$($N,Q\le 10^5$、$A_i\le 10^9$、$k\le 10^{15}$)。

例:$A=[5,3,8]$、$[l,r]=[1,3]$、$k=12$ → 由甜到淡吃 $8+5=13\ge 12$ → $2$ 顆。
:::

::: tip 作法
**先看單次詢問怎麼做**:貪心一定挑最甜的先吃,所以要的是「把區間內的值由大到小排序後,前綴和第一次 $\ge k$ 的位置」。靜態版就是 merge-sort tree;麻煩在**帶修改**——每個節點裡的排序陣列改一個值就要重排,直接掛掉。

**結構:把 merge-sort tree 翻轉過來。** 改建一棵**軸 $=$ 值域**的線段樹(值先離線離散化),每個 value-node 管一段值域;節點內部不再存排序陣列,而是用一個**以索引為軸的 BIT** 存「這段值域內、各索引位置上的 count 與 sum」。離線把每個節點「可能出現過的索引」預先鋪好固定,之後**改值 $=$ 沿新舊兩個值的葉→根路徑,各在對應 BIT 上做 $\pm1/\pm v$**——完全不需要重排。查「值域節點 $u$ 內、索引落在 $[l,r]$ 的 count/sum」就是 BIT 的 `prefix(r) − prefix(l−1)`,任意區間直接相減,**不需要 node 對齊、不用拆區間**(「node 要完全落入查詢區間」那條規則只屬於索引軸的樹)。

**查詢:用 kth-element 式 descent 取代外層二分。** 天真的做法是把「count/sum of 值 $\ge t$」當黑盒、對門檻 $t$ 再包一層二分,$O(\log^3 N)$ 會 TLE。正解是直接**在值域樹上從根走一條 root→leaf 的路**:

每層只問一次「右半(值大那側)在 $[l,r]$ 的 sum」:**夠付剩餘的 $k$ 就走進右半;不夠就把整包吃掉(ans += count、need −= sum),改走左半**;到葉子後剩下的用單一值 $v$ 補,$\lceil \text{need}/v\rceil$ 顆。每一步都跟「由大到小吃」一致。為什麼比 merge-sort tree + 外層二分省一個 log?差在外層二分把 count/sum 當黑盒,每 check 一次就從根重拆一次區間,**搜尋跟樹結構沒共用**;軸對調之後「搜答案」本身就是樹上這趟 descent(每層只進一個 child、不是 DFS),$O(\log^2 N)$/query。二分沒有完全消失——每層還是要在節點的軸上 lower_bound 定位 $l,r$,但那是**查座標**,不是**搜答案**。

**走一遍**:區間值 $\{7,7,5,3,2,2\}$、$k=17$。根($2..7$)sum $=26$ 有解 → 右半($5..7$)sum $=19\ge17$ 進去 → 再右半($7$)sum $=14<17$ 整包吃(ans $=2$、need $=3$)走左 → 葉($5$):$\lceil 3/5\rceil=1$ → ans $=3$ ✓。

**記憶體細節:每個節點要有自己的離散化軸。** 值樹約 $2(N+Q)$ 個節點,若每個節點的 BIT 都開滿 $[1,N]$ 就是上百億格,直接炸。好在**索引 $c$ 一生會有的值(初始 $A_c$ + 之後改到它的所有 $x$)離線都讀得到**:對每對 (索引, 可能值),沿該值的葉→根路徑把索引鋪進節點的軸,每對貢獻 $O(\log)$ 個節點,總格數 $O((N+Q)\log N)$。軸固定不動還有個關鍵好處——**排名不會浮動**:改值只動計數(舊路徑 $-1$、新路徑 $+1$),lower_bound 出來的位置永遠有效。

```cpp
auto tot = qnode(1, l, r);                       // (count, sum)
if (tot.second < k) { print(-1); continue; }
int node = 1; ll need = k, ans = 0;
while (node < SZ) {
    auto rq = qnode(2*node + 1, l, r);           // 右子 = 值大的一半
    if (rq.second >= need) node = 2*node + 1;    // 夠 → 進右子
    else { ans += rq.first; need -= rq.second; node = 2*node; }  // 全吃 → 往左子
}
ans += (need + vals[node - SZ] - 1) / vals[node - SZ];   // 葉:ceil(剩/v)
```

想真正吃透,建議先寫純版「靜態區間第 $k$ 小」——G 就是它加上 BIT 支援改值、多存一條 sum。完整推導、逐步走例與 AC code 在部落格:[ABC467 G Many Sweets Problem 詳解](https://hanayukii.dev/blog/abc467g-many-sweets)。

**另解(讀者提供,已實作驗證)**:反過來以 **index 為外層線段樹、每個節點掛一棵動態開點值域線段樹**——修改沿外層 $O(\log N)$ 個節點各做內層單點加;查詢把 $[l,r]$ 拆成 $O(\log N)$ 個外層節點、收集內層樹根,**多棵同時往值大的那半二分**(帶修區間第 $k$ 小的經典樹套樹,換成維護 sum)。時間同為兩個 log;代價是**空間多一個 log**:實測 $N=Q=10^5$ 約 $3.4	imes10^7$ 個節點(cnt、sum、兩個 child 約 20B/節點)≈ 680MB,1024MB 限制下要用 pool + reserve 壓好才裝得下。本文的離線鋪軸版本質上就是它的靜態化——內層換成 BIT,空間少一個 log。實際提交:**AC,但時間貼著限制**(本機對照約慢 1.6×)——兩版漸進同為 log N·log M,差距全是常數:工作集 80MB vs 700MB 的 cache/TLB 差、動態開點的執行期配置、以及節點資料散在多條陣列的隨機存取。離線鋪軸的本質就是把執行期指標追蹤換成建構期連續記憶體。
