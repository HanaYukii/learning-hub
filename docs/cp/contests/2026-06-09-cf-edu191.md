# Educational Codeforces Round 191 — 學習重點

> 懶人包:每題「題意 + 核心作法」。難度 `~` 為估計,據官方 editorial 整理。

- **[A. AI Project Development](https://codeforces.com/contest/2233/problem/A)** `貪心` `數學` `~900`
  題意:給定整數 n, x, y, z(均 ≤ 10⁴):兩人合寫共 n 行程式的專案,Maxim 從頭到尾每小時寫 x 行;
  Nikita 開工前二選一且不可反悔——從頭每小時寫 y 行,或先花 z 小時架設 AI(期間完全不寫,Maxim 照常寫),之後每小時寫 10y 行。
  兩人合計寫滿 n 行即完工;時間以整小時計(半途完工該小時整算),若架設結束前已寫滿則當場結束。
  輸出:最少幾個整小時完工。
  作法:**答案 = `min(ceil(n, x+y), z + ceil(n − z·x, x+10y))`**,第二項僅在 `z·x ≤ n` 時計入(否則架設完成前 Maxim 已獨力寫完,被第一項支配)。

- **[B. Different Distances](https://codeforces.com/contest/2233/problem/B)** `構造` `~1200`
  題意:給定整數 n(2 ≤ n ≤ 200)。構造長 4n 的整數陣列,使 1..n 每個值恰出現 4 次;
  且對每個值 x,設其四次出現位置依序為 p₁ < p₂ < p₃ < p₄,三個間距 p₂−p₁、p₃−p₂、p₄−p₃ 必須兩兩相異。
  輸出:任一組滿足全部條件的陣列。
  作法:官方依 n 奇偶給閉式 pattern——偶數把值兩兩配對,每對輸出 8 元素塊 `b a a b a b b a`(兩值間距各為 1,2,3 / 3,2,1),奇數先特判前三值再配對。**寶石(評論區):以 n=2 的基底 `[1,2,1,1,2,2,1,2]` 出發,n→n+1 只要把新值插在「舊每個值每次出現之後」即可遞推,免奇偶分類**;解空間極大,random shuffle + 檢查也能過。

- **[C. Cost of a Bracket Sequence](https://codeforces.com/contest/2233/problem/C)** `貪心` `~1400`
  題意:給定括號字串 s(長度 n ≤ 5000,Σn ≤ 5000)與整數 k;定義字串的 cost = 其最長「合法括號序列」子序列的長度。
  要從 s 刪去至多 k 個字元,使剩餘字串的 cost 最小。
  輸出:長 n 的 01 字串,標記每個位置的字元刪或留。
  作法:cost = 2×最大配對數。**寶石:在 balance walk 的全域最低谷 `pos`(最小化 `pref_open[i] + 右側 ')' 數`)切開——左半多餘的全是 '(',右半多餘的全是 ')'**,於是左半從前端貪心標 '(',右半從前端標 ')',用滿 k 個即停。

- **[D. Goods on the Shelf](https://codeforces.com/contest/2233/problem/D)** `構造` `分類討論` `~1800`
  題意:給定長 n 的陣列 a(n ≤ 2×10⁵,值 ≤ 10⁹),aᵢ 為貨架位置 i 上商品的種類;「排列正確」= 每種商品的所有出現位置連成一個連續區塊。
  可至多執行一次操作:選兩個不同位置並交換上面的商品(也可不換)。
  輸出:YES/NO——交換後能否使貨架排列正確。
  作法:某值壞 ⇔ 其出現拆成 ≥2 個區塊。**寶石:一次 swap 只動兩個位置,某值區塊數 > 3 直接 NO;否則 swap 必動第一個壞值的某個區塊邊界**——枚舉該值各區塊端點 ±1 的 O(1) 個候選位置(去重後兩兩互換),每組 O(n) 全檢即可。

- **[E1/E2. Permutation Transmission](https://codeforces.com/contest/2233/problem/E1)** `位元` `排列` `組合計數` `~2100 / ~2400`
  題意:給定 n 與 ⌈log₂(n+1)⌉ 條長 n 的 01 字串,它們原是某個 1..n 排列 p 的 bit 平面——第 j 條是所有 pᵢ 的第 j 個 bit 依位置串成的字串(j 從 0 到 n 的最高位)。
  傳輸後各條字串對應哪個 bit 的順序遺失(以任意順序抵達);資料也可能已損毀,使得不存在與之一致的排列。
  輸出:可能的原排列 p 的個數(可為 0)。E1:n、Σn ≤ 2000;E2:n、Σn ≤ 2×10⁵。
  作法:**寶石:column 值總和 = Σ 2ʲ·(bit j 那條 row 的 1 數),而 n 個相異正整數的和下界 n(n+1)/2 只在 {1..n} 取等 → 由排序不等式,「1 越多的 row 指給越低的 bit」唯一最小化總和**,故只需驗證這一種指派是否產出 1..n;答案 = 各「相同 popcount 群」大小階乘之乘積(同 popcount 互換 = 交換兩個 bit,是雙射,保排列)。E1 官方另解:由高位往低位指派 row 的 3ᵏ 狀態數位 DP(bitset 追蹤仍貼著上界 n 的 column),計數全 column ≤ n 的指派——配合 column 相異且非零,鴿籠即得恰為 1..n。
