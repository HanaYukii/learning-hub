### [A. AI Project Development](https://codeforces.com/contest/2233/problem/A)
`貪心` `數學` `~900`

::: info 題意
給定整數 n, x, y, z(均 ≤ 10⁴):兩人合寫共 n 行程式的專案,Maxim 從頭到尾每小時寫 x 行;Nikita 開工前二選一且不可反悔——從頭每小時寫 y 行,或先花 z 小時架設 AI(期間完全不寫,Maxim 照常寫),之後每小時寫 10y 行。

兩人合計寫滿 n 行即完工;時間以整小時計(半途完工該小時整算),若架設結束前已寫滿則當場結束。

輸出:最少幾個整小時完工。

例:n=20, x=1, y=1, z=2 → 不架 AI:⌈20/2⌉ = 10 小時;架 AI:前 2 小時 Maxim 寫 2 行,剩 18 行以 1+10=11 行/時,⌈18/11⌉ = 2 → 共 4 小時。答案 4。
:::

::: tip 作法
Nikita 只有兩種選擇,各算完工時間取 min 即可:不架 AI 是 `ceil(n, x+y)`;架 AI 是 `z + ceil(n − z·x, x+10y)`,但**只在 `z·x ≤ n` 時才算這一項**——若 Maxim 在架設期間就獨力寫完,完工時間 ≤ z,而這種情況「不架 AI」只會更快(多一個人幫忙),第一項已涵蓋。O(1)。
:::

### [B. Different Distances](https://codeforces.com/contest/2233/problem/B)
`構造` `~1200`

::: info 題意
給定整數 n(2 ≤ n ≤ 200)。構造長 4n 的整數陣列,使 1..n 每個值恰出現 4 次;

且對每個值 x,設其四次出現位置依序為 p₁ < p₂ < p₃ < p₄,三個間距 p₂−p₁、p₃−p₂、p₄−p₃ 必須兩兩相異。

輸出:任一組滿足全部條件的陣列。

例:n=2 → `[1,2,1,1,2,2,1,2]`:值 1 出現於位置 1,3,4,7(間距 2,1,3),值 2 出現於位置 2,5,6,8(間距 3,1,2),皆兩兩相異。
:::

::: tip 作法
構造題,先估解空間有多大:限制只有「每個值的三個間距兩兩相異」,隨便排其實很難全部撞在一起——這暗示解很密,不必執著找唯一 pattern。

官方走閉式 pattern:偶數 n 把值兩兩配對,每對輸出 8 元素塊 `b a a b a b b a`(兩值的間距分別是 1,2,3 與 3,2,1),奇數 n 先特判前三個值再配對。能過,但奇偶 casework 有點繁。

兩條更省事的路:**評論區的遞推**——以 n=2 的基底 `[1,2,1,1,2,2,1,2]` 出發,n→n+1 時把新值插進舊排列即可維持性質,免掉奇偶分類;或者乾脆**隨機化**——n ≤ 200、陣列長 ≤ 800,寫個 O(n) 的檢查函數,random shuffle 到合法為止,期望次數很少。解空間密的構造題,「隨機 + 驗證」常是最快寫完的做法。
:::

### [C. Cost of a Bracket Sequence](https://codeforces.com/contest/2233/problem/C)
`貪心` `~1400`

::: info 題意
給定括號字串 s(長度 n ≤ 5000,Σn ≤ 5000)與整數 k;定義字串的 cost = 其最長「合法括號序列」子序列的長度。

要從 s 刪去至多 k 個字元,使剩餘字串的 cost 最小。

輸出:長 n 的 01 字串,標記每個位置的字元刪或留。

例:s = `(())`, k = 1 → 原 cost 為 4;刪掉開頭的 '(' 剩 `())`,最長合法子序列為 `()`,cost 降至 2,已是 k=1 下的最小。
:::

::: tip 作法
cost = 2 × 最大配對數,所以先把「最大配對數」寫成好操作的形式。**恆等式:最大配對數 = min 過所有切點 i 的(前 i 個字元的 '(' 數 + 第 i 個之後的 ')' 數)**——任何一對配對必然是「切點左邊的 '(' 配切點右邊的 ')'」,所以每個切點都給出一個上界;而在讓這個量最小的切點(balance walk 的全域最低谷),上界恰好取得到。驗證小例:`)(` 在谷底切開得 0 + 0 = 0 ✓。

有了恆等式,刪哪裡就清楚了:**在最低谷左邊刪一個 '(',谷值 −1;右邊刪一個 ')',谷值也 −1;刪別的(左邊的 ')'、右邊的 '(')谷值不動**。而最低谷的定義保證左邊「多餘」的字元就是 '('、右邊多餘的就是 ')',夠刪。所以貪心:找到最低谷切點,左半從前端標 '('、右半標 ')',標滿 k 個或標到沒得標為止。O(n)。

```cpp
// sketch:最低谷切點 + 兩側貪心標刪
int pos = 0, best = INT_MAX;              // pref_open[i]:前 i 字元的 '(' 數
for (int i = 0; i <= n; ++i)              // suf_close[i]:位置 i 之後的 ')' 數
    if (pref_open[i] + suf_close[i] < best)
        best = pref_open[i] + suf_close[i], pos = i;
for (int i = 0; i < pos && k; ++i)
    if (s[i] == '(') del[i] = 1, --k;     // 左半從前端標 '('
for (int i = pos; i < n && k; ++i)
    if (s[i] == ')') del[i] = 1, --k;     // 右半從前端標 ')',用滿 k 即停
```

### [D. Goods on the Shelf](https://codeforces.com/contest/2233/problem/D)
`構造` `分類討論` `~1800`

::: info 題意
給定長 n 的陣列 a(n ≤ 2×10⁵,值 ≤ 10⁹),aᵢ 為貨架位置 i 上商品的種類;「排列正確」= 每種商品的所有出現位置連成一個連續區塊。

可至多執行一次操作:選兩個不同位置並交換上面的商品(也可不換)。

輸出:YES/NO——交換後能否使貨架排列正確。

例:`a = [1, 2, 1, 2]` → 值 1、2 各拆成兩塊;交換位置 2、3 得 `[1, 1, 2, 2]`,兩值皆連成一塊 → YES。
:::

::: tip 作法
先定義「壞」:某值壞 ⇔ 它的出現位置拆成 ≥ 2 個區塊,也就是中間有 gap。一次 swap 只改兩個位置的內容,能修的量有限——**每動一個位置,最多讓某個值消掉一個 gap**,所以一次 swap 至多消 2 個 gap。某值若拆成 3 塊(2 個 gap)還有救:像 `[1,x,1,x,1]` 把中間的 x 跟尾端的 1 互換,一手填 gap、一手縮端點,恰好兩個 gap 一起消;但拆成 **4 塊以上(≥3 個 gap)直接 NO**。

判定怎麼做?swap 若要有效,必然動到**第一個壞值**的某個區塊邊界附近——不然那個值的 gap 原封不動。所以候選位置只有:該值各區塊端點 ±1,常數個。候選兩兩配對互換、每組 O(n) 全檢「所有值是否都連續」,總共 O(常數² · n)。這種「操作次數 = 1」的題,套路就是**先用結構論證把候選壓到 O(1),再暴力驗**。

```cpp
// sketch:第一個壞值的區塊端點 ±1 → 候選集,兩兩互換全檢
vector<int> cand;
for (int e : block_endpoints)              // 該值各區塊的左右端點
    for (int d : {-1, 0, 1})
        if (0 <= e + d && e + d < n) cand.push_back(e + d);
sort(cand.begin(), cand.end());
cand.erase(unique(cand.begin(), cand.end()), cand.end());
for (int i : cand) for (int j : cand) {    // O(1)² 組候選對
    swap(a[i], a[j]);
    if (all_blocks_contiguous(a)) return true;  // O(n) 全檢
    swap(a[i], a[j]);
}
```

### [E1/E2. Permutation Transmission](https://codeforces.com/contest/2233/problem/E1)
`位元` `排列` `組合計數` `~2100 / ~2400`

::: info 題意
給定 n 與 ⌈log₂(n+1)⌉ 條長 n 的 01 字串,它們原是某個 1..n 排列 p 的 bit 平面——第 j 條是所有 pᵢ 的第 j 個 bit 依位置串成的字串(j 從 0 到 n 的最高位)。

傳輸後各條字串對應哪個 bit 的順序遺失(以任意順序抵達);資料也可能已損毀,使得不存在與之一致的排列。

輸出:可能的原排列 p 的個數(可為 0)。E1:n、Σn ≤ 2000;E2:n、Σn ≤ 2×10⁵。

例:n=3,收到兩條字串 `011` 與 `110` → 指派 (bit 0, bit 1) = (`011`, `110`) 逐 column 重建得 p = [2,3,1];對調指派得 [1,3,2],皆為 1..3 的排列 → 答案 2。
:::

::: tip 作法
把「指派 rows → bits」後重建的值寫下來:column i 的值 = Σⱼ 2ʲ · rowⱼ[i],要求這 n 個值恰為 1..n。指派方式有 B! 種,直接枚舉不可行,先看**總和**:

$$\sum_i \mathrm{val}_i=\sum_j 2^j\cdot(\text{row}_j\ \text{的 1 數}).$$

關鍵引理:**n 個相異正整數的和 ≥ n(n+1)/2,取等 ⟺ 恰為 {1..n}**。而由排序不等式,「1 越多的 row 指給越低的 bit(權重 2ʲ 越小)」是唯一讓總和最小的指派——若連這個最小總和都 > n(n+1)/2,答案 0;等於的話,再驗 columns 相異且非零,就恰好是排列。**所以只需要驗證這一種指派**,B! 的枚舉塌成一次重建 + 檢查。

計數:同 popcount 的 rows 互換,總和不變;互換 = 對每個值做「交換兩個 bit」的映射,這是單射(值仍兩兩相異)且保零(值仍非零),總和又是 n(n+1)/2 → 由引理**仍然是 1..n 的排列**。不同 popcount 互換則總和變大、必爆。所以答案 = 各「相同 popcount 群」大小的階乘之乘積。

E1 官方另解是由高位往低位指派的 3ᵏ 狀態數位 DP(bitset 追蹤仍貼上界 n 的 column),但排序不等式這條路 E2 的 2×10⁵ 也直接過。

```cpp
// sketch:「1 越多的 row → 越低的 bit」唯一候選 + 驗證計數
sort(rows.begin(), rows.end(),
     [&](auto& A, auto& B) { return ones(A) > ones(B); });  // rows[j] → bit j
for (int i = 0; i < n; ++i) {
    long long v = 0;
    for (int j = 0; j < B; ++j)
        v |= (long long)(rows[j][i] - '0') << j;  // 逐 column 重建值
    val[i] = v;
}
if (!is_perm_1n(val)) ans = 0;                    // 唯一候選不合法 → 0
else { ans = 1;
       for (int c : same_popcount_group_sizes)    // 同 popcount 可互換
           ans = ans * fact[c] % MOD; }
```
