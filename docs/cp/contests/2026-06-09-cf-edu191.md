# Educational Codeforces Round 191 — 學習重點

> 懶人包:每題「技巧 + 作法」。各難度都收主流可遷移的;偏門/太難略過。難度 `~` 為估計,據官方 editorial 整理。

- **[A. AI Project Development](https://codeforces.com/contest/2233/problem/A)** `貪心` `數學` `閉式` `~900`
  作法:兩種推進方式——純做工每天 `x+y`,或先固定花 `t` 天蓄能後續每天 `x+10y`(`t` 是輸入給定,非自由枚舉)。**答案 = `min(ceil(n, x+y), ceil(n − t·x, x+10y) + t)`**,第二項僅在 `t·x ≤ n` 時成立。

- **[B. Different Distances](https://codeforces.com/contest/2233/problem/B)** `構造` `奇偶分類` `~1200`
  作法:把 1..n 各 4 份排出,使每個值三段間距互異。官方解按 `n` 奇偶各給一條閉式 pattern。**寶石(評論區):以 n=2 的基底 `[1,2,1,1,2,2,1,2]` 出發,n→n+1 只要把新值插在「舊每個值每次出現之後」即可遞推,免奇偶分類**;random shuffle 暴力檢查也能過(解空間極大)。

- **[C. Cost of a Bracket Sequence](https://codeforces.com/contest/2233/problem/C)** `貪心` `前綴平衡` `~1400`
  作法:cost = 2×可配對數,要最小化配對。**寶石:在 balance walk 的全域最低谷 `pos`(最小化 `pref_open[i] + 剩餘')'`)切開——左半多餘的全是 '(',右半多餘的全是 ')'。** 於是左半從前端貪心標 k 個 '(',右半從前端起標 ')',用滿 k 個即停。

- **[D. Goods on the Shelf](https://codeforces.com/contest/2233/problem/D)** `構造` `分類討論` `貪心觀察` `~1800`
  作法:目標每個值連成一段。某值「壞」⇔ `max_idx − min_idx + 1 ≠ 出現次數`。**寶石:一次 swap 至多影響 2 個值的區塊,故壞區塊數 > 3 直接 NO;** 否則鎖定第一個壞值,必動到它的 min 或 max,只枚舉各值區塊邊界附近 O(1) 個候選位置做互換驗證(實作上把候選位置去重後 O(候選²) 暴力試)。

- **[E1/E2. Permutation Transmission](https://codeforces.com/contest/2233/problem/E1)** `位元/popcount` `排列充要條件` `計數` `~2100 / ~2400`
  作法:用 k≈⌊log2 n⌋+1 條 bit-row,使各 column 組成 1..n 的排列。**寶石:無重複正整數序列是 1..n 排列 ⇔ 總和 = n(n+1)/2,而這正是和的下界 → 把 row 依「1 的個數」由多到少排序,唯一最小化各 column 值的總和。** 第 j 個 bit 取第 j 重的 row;驗證 column 恰為 1..n,答案 = 各「相同 popcount 群」大小階乘之乘積(同 popcount 可任意互換)。E2 即此閉式;E1 為小範圍版,改用 bitset + 3 進制狀態 DP 同思路驗證。

**不收**:~~F. Shortest GCD Paths~~ → 約化後在 `(a/g, b/g)` 因數格上跑記憶化 DP(轉移成本 `max(從 a 拿掉的因數, 往 b 加的因數)`),含「最優轉移至多一側動 ≥2 質因數」的成本拆解論證;技巧雖可遷移(因數 DAG 最短路)但細節吃實作與數論直覺、估計 ~2700,短期投報率低。
