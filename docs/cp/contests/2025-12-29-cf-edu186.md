### [A. New Year String](https://codeforces.com/contest/2182/problem/A)
`字串` `分類討論` `~800`

::: info 題意
定義:字串是 New Year 字串 ⇔ **含連續子串 `2026`,或不含連續子串 `2025`**(滿足其一即可)。

給定字串 $s$,每次操作可把任一字元改成 `0`/`2`/`5`/`6` 之一。**求**:使 $s$ 成為 New Year 字串的最少操作數。

例:`s = 2025` → 含 `2025` 且無 `2026`,把末位 `5`→`6` 得 `2026` → $1$ 次;`s = 000` 不含 `2025` → $0$ 次。
:::

::: tip 作法
答案只有 $0/1$:已合法輸出 $0$;否則 $s$ 必含 `2025`,**把其中的 `5` 就地改成 `6`,同一位置直接製造出 `2026`** → $1$。
:::

### [B. New Year Cake](https://codeforces.com/contest/2182/problem/B)
`貪心` `模擬` `~1100`

::: info 題意
蛋糕由上而下第 $t$ 層大小為 $2^{t-1}$,相鄰層黑白巧克力必須交錯(頂層顏色自選),大小 $k$ 的層耗 $k$ 公斤該色。

給定白 $a$ 公斤、黑 $b$ 公斤,**求**最大層數(至少 $1$ 層)。

例:$a=6,b=2$ → $3$ 層($1,2,4$):底層色吃 $4+1=5\le 6$、另色吃 $2\le 2$;想疊 $4$ 層時底層色要 $8+2=10>6$ → 答案 $3$。
:::

::: tip 作法
兩色各吃一條等比($1+4+16+\cdots$ 與 $2+8+32+\cdots$);**$k$ 層可行 ⇔ $\max(a,b)\ge$ 底層色的和、$\min(a,b)\ge$ 另一和**(底層和恆較大 → 大存量配底層必最優),和以 $4^t$ 爆炸、層數 $O(\log)$,由 $k=1$ 遞增模擬即可。
:::

### [C. Production of Snowmen](https://codeforces.com/contest/2182/problem/C)
`不變量` `模擬` `~1500`

::: info 題意
三條循環輸送帶各 $n$ 顆雪球,大小分別為 $a_i,b_i,c_i$(下標模 $n$ 循環)。選起點 $(i,j,k)$ 後,第 $t$ 個雪人($t=0,\dots,n-1$)用頭 $a_{i+t}$、身 $b_{j+t}$、腳 $c_{k+t}$。

雪人穩定 ⇔ 頭 $<$ 身 $<$ 腳(嚴格)。**求**:使全部 $n$ 個雪人都穩定的 $(i,j,k)$ 組數。

例:$n=2$,$a=[1,3]$,$b=[2,4]$,$c=[3,5]$ → 對齊的 $(1,1,1)$、$(2,2,2)$:$1<2<3$、$3<4<5$ ✓;一旦錯位就出現 $a_2=3\ge b_1=2$ ✗ → 答案 $2$。
:::

::: tip 作法
**合法性只依賴相對位移**:$(i,j)$ 合法與否只看 $s_1=(i-j)\bmod n$、$(j,k)$ 只看 $s_2=(j-k)\bmod n$,兩位移獨立、絕對起點自由 → **答案 $=(\text{好位移數})_{ab}\times(\text{好位移數})_{bc}\times n$**;每個位移 $O(n)$ 驗證,共 $O(n^2)$。

```cpp
// sketch:枚舉相對位移
auto cnt = [&](vector<int>& hi, vector<int>& lo) { // # 位移 s:hi[t] > lo[(t+s)%n] ∀t
    int c = 0;
    for (int s = 0; s < n; ++s) {
        bool ok = true;
        for (int t = 0; t < n; ++t) ok &= (hi[t] > lo[(t + s) % n]);
        c += ok;
    }
    return c;
};
long long ans = 1LL * cnt(b, a) * cnt(c, b) * n;
```

### [D. Christmas Tree Decoration](https://codeforces.com/contest/2182/problem/D)
`組合計數` `貪心` `~2000`

::: info 題意
$n$ 人、$n{+}1$ 箱(編號 $0..n$),箱 $i$ 有 $a_i$ 個裝飾($n\le 50$)。給定排列 $p$ 後依 $p_1,\dots,p_n,p_1,\dots$ 輪流,每人輪到時從 **$0$ 號箱或自己編號的箱**取一個裝飾掛上,直到全部掛完。

若輪到某人時兩箱皆空(但裝飾還沒掛完)即失敗;排列公平 ⇔ 存在取法全程不失敗。**求**:公平排列數 $\bmod\ 998244353$。

例:$n=2$,$a=[1,3,1]$,共 $5$ 個 → 輪數 $k=2$、多出的第 $5$ 次由 $p_1$ 掛。$1$ 號箱有 $3$ 個、只有本人清得掉 → $1$ 號必須排頭多掛一次;驗 $[1,2]$ 可行、$[2,1]$ 時 $1$ 號箱剩一個沒人掛 → 答案 $1$。
:::

::: tip 作法
設總數 $T$、$k=\lfloor T/n\rfloor$、$r=T\bmod n$:排列前 $r$ 席掛 $k{+}1$ 次、其餘 $k$ 次。可行性**與順序幾乎無關**:每人先吃自己箱 $\min(a_i,k)$、不足向 $0$ 號箱補——$0$ 號箱補到負、或有箱剩 $>1$(誰也清不掉)→ 答案 $0$。

剩恰 $1$ 的 $w$ 人**必須佔前 $r$ 席**(尾輪清自己箱);$0$ 號箱餘 $x_0$ 個,由箱已清空的 $z$ 人中挑 $x_0$ 人補滿前 $r$ 席($r=w+x_0$)→ 答案 **$z!\cdot(w+x_0)!/x_0!$**($=\binom{z}{x_0}\,r!\,(n-r)!$)。

```cpp
// sketch:k 輪吃滿 + 尾輪安置
int k = accumulate(a.begin(), a.end(), 0) / n, z = 0;  bool bad = false;
for (int i = 1; i <= n; ++i) {
    int x = min(a[i], k);
    a[i] -= x;  a[0] -= k - x;        // 先吃自己箱,不足找 0 號箱
    z += (a[i] == 0);
    bad |= (a[i] > 1 || a[0] < 0);    // 剩 >1 沒人清得掉 / 0 號箱透支
}
int w = n - z;                        // 剩恰 1 的人:必佔前 r 席
// 答案 = bad ? 0 : z! * (w + a[0])! / a[0]!    (r = w + a[0])
```

### [E. New Year's Gifts](https://codeforces.com/contest/2182/problem/E)
`貪心` `資料結構` `~1900`

::: info 題意
$n$ 個朋友、$m$ 個盒(盒 $i$ 美觀 $a_i$,每盒至多裝一件、用盒免費)、總預算 $k$。給朋友 $i$ 的禮物須值 $\ge y_i$(花費 = 價值),他會開心 ⇔ **盒美觀 $\ge x_i$ 或 禮物值 $\ge z_i$**($z_i>y_i$)。

每人都必須買禮物(可不裝盒)。**求**:最多能讓幾個朋友開心。

例:$k=5$、一個美觀 $10$ 的盒;甲 $(x{=}3,y{=}1,z{=}4)$、乙 $(x{=}2,y{=}1,z{=}100)$。付底價 $1{+}1$ 後剩 $3$:盒配升級費最貴的乙(免費開心),甲補 $z-y=3$ 元 → $2$ 人;反之盒給甲,乙差 $99$ 補不起 → 只 $1$ 人。
:::

::: tip 作法
先付所有底價 $\sum y_i$、剩餘預算 $k'$;朋友 $i$ 的「用錢開心」費 $=z_i-y_i$。**美觀由小到大掃描:遇到盒就配給目前合資格($x_i\le$ 盒美觀)中升級費最大的朋友**(multiset 取 max)——這使殘留費用集合 pointwise 最小;最後剩下的人由便宜到貴能付就付。

```cpp
// sketch:掃描 + multiset 交換貪心
multiset<long long> s;                          // 尚未開心者的升級費 z-y
for (v : 美觀值由小到大) {
    for (朋友 i : x[i] == v) s.insert(z[i] - y[i]);          // 先進場(x_i ≤ v 合格)
    for (盒 : 美觀 == v)
        if (!s.empty()) { s.erase(prev(s.end())); ++ans; }   // 盒救最貴的
}
for (long long c : s) { if (k2 < c) break; k2 -= c; ++ans; } // 剩的由便宜到貴買
```

### [F1/F2. Christmas Reindeer](https://codeforces.com/contest/2182/problem/F2)
`組合計數` `位元` `數位dp` `~2300`

::: info 題意
每隻鹿的力量都是 $2$ 的冪 $2^{c_i}$。一組鹿的載重:力量值降序排成 $s_1\ge s_2\ge\cdots\ge s_k$,載重 $=s_1+\lfloor s_2/2\rfloor+\lfloor s_3/4\rfloor+\cdots+\lfloor s_k/2^{k-1}\rfloor$。

處理 $m$ 筆操作:加入一隻 $2^x$、移除一隻 $2^x$,或**詢問**:選當前鹿群子集(同力量的鹿視為不同隻)使載重 $\ge x$ 的方案數 $\bmod\ 998244353$。F1:$n,m\le 500$;F2:$n,m\le 3\cdot 10^5$(指數與詢問值都在 60-bit 內)。

例(題面):群 = 兩隻力量 $1$(指數 $0$),詢問 $x=1$:單選任一隻載重 $1$、全選 $1+\lfloor 1/2\rfloor=1$ → $3$ 種。
:::

::: tip 作法
降序第 $t$ 個(從 $0$ 數)被選的鹿貢獻 $\lfloor 2^{e-t}\rfloor$ → **$x$ 的標準需求:由高往低第 $t$ 個 set bit(位置 $i$)需要一隻指數恰 $i+t$ 的鹿**(如 $x=3=11_2$ → 需兩隻指數 $1$:$2+\lfloor 2/2\rfloor=3$);又**多選鹿載重不減**,故達標 ⇔ 各指數選取量由高往低逐位「不輸」需求。

仿數位 DP 按**第一個嚴格超過需求的指數**分類:更高位恰好($\prod\binom{\mathrm{freq}_i}{\mathrm{req}_i}$)× 該位選超過 $\mathrm{req}_i$ 隻 × 更低位全自由($2^{\#}$)。加刪只動 freq 桶;桶僅 $61$ 個、每詢問 $O(61^2)$,同一份解通吃 F1/F2。

```cpp
// sketch:詢問 x —— 按第一個超過需求的指數分類
auto req = required(x);          // 由高往低第 t 個 set bit(位 i)→ 需指數 i+t 一隻
int ans = 0, cur = 1, below = 總鹿數;
for (int i = 60; i >= 0; --i) {
    below -= freq[i];
    ans = (ans + 1LL * cur * waysMoreThan(freq[i], req[i]) % MOD * pow2[below]) % MOD;  // 此位超過,低位自由
    cur = 1LL * cur * C(freq[i], req[i]) % MOD;   // 此位也恰好,續比
}
ans = (ans + cur) % MOD;         // 每位都恰取需求 → 恰好達標
```
:::

例:星形樹(根 + 兩葉)、$k=2$:燈 $2$ 放任一葉、燈 $3$ 放另一葉(跳距 $2\le k$)→ $2$ 種;$k=1$ 時第二跳距離 $2>1$ → $0$ 種。

::: tip 作法
「必選最深」⇒ **擺法就是受限 DFS:一進子樹必走完;在 $v$ 處只有「最後一個孩子」的收尾往外延伸,其餘孩子須在距 $v\le k-1$ 處收尾**(跳往下個兄弟距離才 $\le k$),且其餘孩子順序自由 → 乘 $(c-1)!$。設 $\mathrm{dp}[v][j]=$ 走完子樹 $v$、收尾點距 $v\le j$ 的方案數,枚舉最後孩子合併。

實作寶石:**整條繼承最深孩子的 dp 陣列 + affine lazy tag(整列乘/加 $O(1)$),其餘孩子逐項併入**——陣列長 = 子樹高,$\sum$ 非最深高度 $=O(n)$(長鏈式合併),整體近線性(另加逆元的 log)。

```cpp
// sketch:枚舉最後孩子 + 高度合併
void dfs(int v) {
    for (int u : g[v]) { dfs(u); shift(dp[u]); }   // 距離基準 u → v(整體 +1)
    // val[u] = dp[u].get(k-1):u 子樹收尾距 v ≤ k-1(跳出去 ≤ k)
    // u 當最後孩子的權重 cur[u] = (c-1)! * Π_{w≠u} val[w]
    dp[v] = inherit(dp[deepest]);                  // 整條搬最深孩子,乘 cur 用 lazy
    for (int u : g[v]) if (u != deepest) {
        addAll(dp[v], cur[u] * dp[u].get(n));      // lazy 整列 +:j 超過 u 高度後全數收錄
        for (int j = 0; j < len(dp[u]); ++j)       // 淺段再逐項修正回「收尾距 ≤ j」
            dp[v][j] += cur[u] * (dp[u].get(j) - dp[u].get(n));
    }
}
// 答案 = dp[root].get(n)
```
