### [E. Digit Circus](https://atcoder.jp/contests/abc465/tasks/abc465_e)
`數位dp` `狀壓` `~1800`

::: info 題意
給定十進位整數 N(1≤N<10⁵⁰⁰,以字串讀入)。

三個條件:(1) x 是 3 的倍數;(2) x 的十進位表示含數字 3;(3) x 的十進位表示恰使用 3 種相異數字(表示皆不含前導零)。

輸出:1≤x≤N 中「恰滿足其中一個條件」的 x 的個數,mod 998244353。

例:`N=13` → 6、9、12 只滿足「3 的倍數」,13 只滿足「含數字 3」→ 答案 4;3 因同時滿足 (1)(2) 兩條件,不計。
:::

::: tip 作法
數位 DP,狀態 = 已用數字集合(bitmask 2¹⁰)× mod 3 餘數 × 是否貼緊 N;末端對每個 (mask, m) 檢查三條件恰一成立。

**leading zero 不可算成用過數字 0** 是關鍵坑。O(2¹⁰·10·|N|),|N| 為位數(≤500)。

```cpp
// sketch:數位 DP 轉移 —— 狀態 (mask, m, tight)
for (int i = 0; i < len; ++i)
    for (auto& [st, cnt] : dp) {              // st = (mask, m, tight)
        auto [mask, m, tight] = st;
        int hi = tight ? s[i] - '0' : 9;
        for (int d = 0; d <= hi; ++d) {
            bool lead = (mask == 0 && d == 0);        // 仍在前導零
            int nmask = lead ? 0 : mask | (1 << d);   // 前導零不算用過 0
            ndp[{nmask, (m + d) % 3, tight && d == hi}] += cnt;
        }
    }
// 末端:A=(m==0)、B=mask>>3&1、C=popcount(mask)==3
// A+B+C==1 才計入;mask==0(即 x=0)不計
```
:::

### [F. Sjeltzer?](https://atcoder.jp/contests/abc465/tasks/abc465_f)
`容斥` `差分` `~2200`

::: info 題意
給定 N(1≤N≤3×10⁵)瓶飲料,第 i 瓶有 6 位數字字串 ID S_i(兩兩相異)與整數大小 V_i(1≤V_i≤10⁹)。

接著 Q(1≤Q≤3×10⁵)筆詢問,每筆給兩個 6 位數字字串 x,y,求所有 ID s 滿足「對每一位 k=1..6 都有 x_k≤s_k≤y_k」(逐位比較的 6 維盒,不是數值區間)之飲料的 V 總和。

輸出:每筆詢問一行答案。

例:三瓶 ID/V 為 `111111`/5、`121212`/7、`120000`/100,詢問 `x=111111`、`y=121212` → 前兩瓶每一位都落在 [x_k,y_k] 內,計入;`120000` 數值上介於其間,但第 3 位 0∉[1,1],排除 → 答案 5+7=12。
:::

::: tip 作法
把 ID 視為 10⁶ 格點,**先逐維由小到大累加做 6 維前綴和(zeta 變換),查詢再對 6 個下界做 2⁶ 容斥**把盒子拆成前綴和之差。

前處理 O(6·10⁶),每詢問 O(2⁶·6);某位 x_k>y_k 直接答 0。

```cpp
// sketch:6 維前綴和(zeta)+ 查詢 2^6 容斥
for (int d = 0, p = 1; d < 6; ++d, p *= 10)      // 逐維、由小到大累加
    for (int v = 0; v < 1000000; ++v)
        if (v / p % 10) f[v] += f[v - p];        // f 變成 6 維前綴和
long long ans = 0;
for (int m = 0; m < 64; ++m) {                   // 對 6 個下界容斥
    int id = 0, p = 1; bool out = false;
    for (int k = 5; k >= 0; --k, p *= 10) {      // k = 字串位(低位在後)
        int dig = (m >> k & 1) ? x[k] - '1' : y[k] - '0';  // 下界側取 x_k−1
        if (dig < 0) { out = true; break; }
        id += dig * p;
    }
    if (!out) ans += (__builtin_popcount(m) & 1 ? -1 : 1) * f[id];
}
```
:::

### [G. Sum of Mex of Mod of Linear](https://atcoder.jp/contests/abc465/tasks/abc465_g)
`ODT` `數論` `~3500`

::: info 題意
給定整數 N,M,C,K(1≤N≤2×10⁵、0≤C<M≤10⁹、1≤K≤10⁹)與長度 N 的序列 A(0≤A_i<M)。

接著 Q(1≤Q≤2×10⁵)筆詢問,每筆給 (i,X),先把 A_i 永久改成 X(0≤X<M),再求 Σ_{k=0}^{K−1} mex{(Ck+A_j) mod M | 1≤j≤N},其中 mex 為集合未含的最小非負整數。

輸出:每筆詢問一行答案。

例:`M=4, C=1, K=2, A=(0,1)` → k=0:mex{0,1}=2;k=1:集合為 {1,2},mex=0(缺 0)→ 答案 2+0=2。
:::

::: tip 作法
**固定 k 時 mex = 自 x=(−Ck) mod M 起連續屬於 A 的 run 長 dₓ**,故答案 = Σ cₓ·dₓ(cₓ = 對應到 x 的 k 個數,查詢間不變)。
