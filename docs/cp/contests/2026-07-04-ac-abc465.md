# AtCoder Beginner Contest 465 — 學習重點

> 懶人包:每題「技巧 + 作法」。各難度都收主流可遷移的;偏門/太難略過。難度 `~` 為 CF-equivalent。
>
> A–B 熱身略過(比例判斷 `3A>2B`、逐時段累加費用)。從 C 開始列。

- **[C. Reverse Permutation](https://atcoder.jp/contests/abc465/tasks/abc465_c)** `模擬` `~800`
  作法:逐一把 k=1..N 接到序列尾端,遇 S_k='o' 就反轉整串。**別真的每次反轉(O(N²)),只記反轉次數奇偶**:偶數 append 尾端、奇數 prepend 前端(deque/雙陣列),最後若總次數為奇再整體反轉一次。O(N)。

- **[D. X to Y](https://atcoder.jp/contests/abc465/tasks/abc465_d)** `樹` `數學` `~1200`
  作法:操作等價於 K 進位下「刪末位數字 / 補一個末位數字」。**每個整數是 K 元 trie 上的節點,轉換步數 = 兩點樹距離(LCA 型)**。答案 = X、Y 的 K 進位長度各扣掉公共前綴長 p:(len_X−p)+(len_Y−p);或直接 while 迴圈把較大者整除 K 累加。

- **[E. Digit Circus](https://atcoder.jp/contests/abc465/tasks/abc465_e)** `數位dp` `狀壓` `~1800`
  作法:**數位 DP**,狀態 = 已用數字集合(bitmask 2¹⁰)× mod 3 餘數 × 是否貼緊上界 N。逐位轉移計數;**leading zero 不可算成用過數字 0** 是關鍵坑。末端統計「用滿 3 種數字 / 含數字 3 / 被 3 整除」**恰一項成立**者。O(2¹⁰·10·log N)。

- **[F. Sjeltzer?](https://atcoder.jp/contests/abc465/tasks/abc465_f)** `容斥` `差分` `~2200`
  作法:每筆 ID 是 6 位(base 10)向量,查詢求「各位 x_k ≤ s_k ≤ y_k」六維盒子內 value 總和。**先做 6 維前綴和(逐維由小到大累加的 zeta 變換 = 高維前綴和),再對 6 個下界做 2⁶ 容斥**,把盒子拆成前綴和之差。前處理 O(D·N + Bᴰ·D),每詢問 O(2ᴰ·D),D=6。

- **[G. Sum of Mex of Mod of Linear](https://atcoder.jp/contests/abc465/tasks/abc465_g)** `ODT` `數論` `~3500`
  作法(超 ~2700,破例作 floor sum + ODT 模板題):答案 = Σₓ cₓ·dₓ,cₓ = 滿足 Ck+x≡0 (mod M) 的 k 個數(查詢間不變),dₓ = 從 x 起連續落在集合 A 的最長長度(≤N)。**用 ODT 把 A 當圓環上的極大區間管理**,每次修改只動 O(1) 個區間,差分寫成 c 的區間和,再用 **floor sum / 一般化 floor sum** 快速求。注意 A 為全集或恰缺一值要特判。
