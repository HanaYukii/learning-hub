# AtCoder Beginner Contest 465 — 學習重點

> 懶人包:每題「題意 + 核心作法」。難度 `~` 為 CF-equivalent。
>
> A–B 熱身略過(比例判斷 `3A>2B`、逐時段累加費用)。從 C 開始列。

- **[C. Reverse Permutation](https://atcoder.jp/contests/abc465/tasks/abc465_c)** `模擬` `~800`
  題意:給定整數 N(2≤N≤5×10⁵)與長度 N、由 `o`/`x` 組成的字串 S;另有整數序列 A,初始 A=(1,2,…,N)。依 k=1,2,…,N 的順序操作:若 S_k=`o` 就反轉 A 的前 k 項(A←(A_k,…,A_1,A_{k+1},…,A_N)),若 S_k=`x` 則不動。
  輸出:所有操作結束後的序列 A(空白分隔)。
  作法:**「反轉前 k 項」等價於「把 k 接到已處理序列尾端後反轉整串」**(前 k 項恰為已處理元素);不真反轉、只記反轉次數奇偶——偶 append 尾、奇 prepend 頭(雙陣列/deque),最後總次數為奇再整體反轉一次。O(N)。

- **[D. X to Y](https://atcoder.jp/contests/abc465/tasks/abc465_d)** `樹` `數學` `~1200`
  題意:給定整數 X,Y(0≤X,Y≤10¹⁸)與 K(2≤K≤10¹⁸)。變數 x 初始為 X,每次操作可選任一整數 y 滿足 ⌊x/K⌋=y 或 ⌊y/K⌋=x,並把 x 換成 y。
  輸出:使 x 變為 Y 的最少操作次數;共 T(1≤T≤2×10⁵)筆測資,各一行作答。
  作法:操作即 K 進位「刪末位 / 補末位」——**每個整數是 K 元 trie 上的節點,答案 = 兩點樹距離(LCA 型)**:(len_X−p)+(len_Y−p),p 為 K 進位公共前綴長;實作直接 while(x≠y) 把較大者 /=K 計數即可。

- **[E. Digit Circus](https://atcoder.jp/contests/abc465/tasks/abc465_e)** `數位dp` `狀壓` `~1800`
  題意:給定十進位整數 N(1≤N<10⁵⁰⁰,以字串讀入)。三個條件:(1) x 是 3 的倍數;(2) x 的十進位表示含數字 3;(3) x 的十進位表示恰使用 3 種相異數字(表示皆不含前導零)。
  輸出:1≤x≤N 中「恰滿足其中一個條件」的 x 的個數,mod 998244353。
  作法:數位 DP,狀態 = 已用數字集合(bitmask 2¹⁰)× mod 3 餘數 × 是否貼緊 N;末端對每個 (mask, m) 檢查三條件恰一成立。**leading zero 不可算成用過數字 0** 是關鍵坑。O(2¹⁰·10·|N|),|N| 為位數(≤500)。

- **[F. Sjeltzer?](https://atcoder.jp/contests/abc465/tasks/abc465_f)** `容斥` `差分` `~2200`
  題意:給定 N(1≤N≤3×10⁵)瓶飲料,第 i 瓶有 6 位數字字串 ID S_i(兩兩相異)與整數大小 V_i(1≤V_i≤10⁹)。接著 Q(1≤Q≤3×10⁵)筆詢問,每筆給兩個 6 位數字字串 x,y,求所有 ID s 滿足「對每一位 k=1..6 都有 x_k≤s_k≤y_k」(逐位比較的 6 維盒,不是數值區間)之飲料的 V 總和。
  輸出:每筆詢問一行答案。
  作法:把 ID 視為 10⁶ 格點,**先逐維由小到大累加做 6 維前綴和(zeta 變換),查詢再對 6 個下界做 2⁶ 容斥**把盒子拆成前綴和之差。前處理 O(6·10⁶),每詢問 O(2⁶·6);某位 x_k>y_k 直接答 0。

- **[G. Sum of Mex of Mod of Linear](https://atcoder.jp/contests/abc465/tasks/abc465_g)** `ODT` `數論` `~3500`
  題意:給定整數 N,M,C,K(1≤N≤2×10⁵、0≤C<M≤10⁹、1≤K≤10⁹)與長度 N 的序列 A(0≤A_i<M)。接著 Q(1≤Q≤2×10⁵)筆詢問,每筆給 (i,X),先把 A_i 永久改成 X(0≤X<M),再求 Σ_{k=0}^{K−1} mex{(Ck+A_j) mod M | 1≤j≤N},其中 mex 為集合未含的最小非負整數。
  輸出:每筆詢問一行答案。
  作法:**固定 k 時 mex = 自 x=(−Ck) mod M 起連續屬於 A 的 run 長 dₓ**,故答案 = Σ cₓ·dₓ(cₓ = 對應到 x 的 k 個數,查詢間不變)。**用 ODT 把 A 當圓環上的極大區間管理**,每次修改只動 O(1) 個區間,差分寫成 c 的區間和,用 **floor sum / 一般化 floor sum** 快速求;A 為全集或恰缺一值要特判。
