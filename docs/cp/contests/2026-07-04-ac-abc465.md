# AtCoder Beginner Contest 465 — 學習重點

> 懶人包:每題「題意 + 核心作法」。各難度都收主流可遷移的;偏門/太難略過。難度 `~` 為 CF-equivalent。
>
> A–B 熱身略過(比例判斷 `3A>2B`、逐時段累加費用)。從 C 開始列。

- **[C. Reverse Permutation](https://atcoder.jp/contests/abc465/tasks/abc465_c)** `模擬` `~800`
  題意:初始 A=(1,…,N)(N≤5×10⁵),依 k=1..N:若 S_k='o' 就反轉 A 的前 k 項,輸出最終 A。
  作法:**「反轉前 k 項」等價於「把 k 接到已處理序列尾端後反轉整串」**(前 k 項恰為已處理元素);不真反轉、只記反轉次數奇偶——偶 append 尾、奇 prepend 頭(雙陣列/deque),最後總次數為奇再整體反轉一次。O(N)。

- **[D. X to Y](https://atcoder.jp/contests/abc465/tasks/abc465_d)** `樹` `數學` `~1200`
  題意:給 X,Y≤10¹⁸、K≥2(T≤2×10⁵ 組),每次操作可把 x 換成 ⌊x/K⌋ 或任一滿足 ⌊y/K⌋=x 的 y,求 x 從 X 變成 Y 的最少操作數。
  作法:操作即 K 進位「刪末位 / 補末位」——**每個整數是 K 元 trie 上的節點,答案 = 兩點樹距離(LCA 型)**:(len_X−p)+(len_Y−p),p 為 K 進位公共前綴長;實作直接 while(x≠y) 把較大者 /=K 計數即可。

- **[E. Digit Circus](https://atcoder.jp/contests/abc465/tasks/abc465_e)** `數位dp` `狀壓` `~1800`
  題意:給 N<10⁵⁰⁰,求 1..N 中「是 3 的倍數 / 十進位含數字 3 / 恰使用 3 種相異數字」三條件**恰滿足一個**的整數個數,mod 998244353。
  作法:數位 DP,狀態 = 已用數字集合(bitmask 2¹⁰)× mod 3 餘數 × 是否貼緊 N;末端對每個 (mask, m) 檢查三條件恰一成立。**leading zero 不可算成用過數字 0** 是關鍵坑。O(2¹⁰·10·|N|),|N| 為位數(≤500)。

- **[F. Sjeltzer?](https://atcoder.jp/contests/abc465/tasks/abc465_f)** `容斥` `差分` `~2200`
  題意:N≤3×10⁵ 個相異 6 位數字串 ID 各帶權 V_i;Q≤3×10⁵ 筆詢問給 6 位串 x,y,求**每一位獨立**滿足 x_k≤s_k≤y_k 的 ID 權值和(逐位比較的 6 維盒,不是數值區間)。
  作法:把 ID 視為 10⁶ 格點,**先逐維由小到大累加做 6 維前綴和(zeta 變換),查詢再對 6 個下界做 2⁶ 容斥**把盒子拆成前綴和之差。前處理 O(6·10⁶),每詢問 O(2⁶·6);某位 x_k>y_k 直接答 0。

- **[G. Sum of Mex of Mod of Linear](https://atcoder.jp/contests/abc465/tasks/abc465_g)** `ODT` `數論` `~3500`
  題意:給 0≤C<M≤10⁹、K≤10⁹、N,Q≤2×10⁵;每筆詢問把 A_i 改成 X 後,求 Σ_{k=0}^{K−1} mex{(Ck+A_j) mod M}。
  作法(超 ~2700,破例收作 floor sum + ODT 模板題):**固定 k 時 mex = 自 x=(−Ck) mod M 起連續屬於 A 的 run 長 dₓ**,故答案 = Σ cₓ·dₓ(cₓ = 對應到 x 的 k 個數,查詢間不變)。**用 ODT 把 A 當圓環上的極大區間管理**,每次修改只動 O(1) 個區間,差分寫成 c 的區間和,用 **floor sum / 一般化 floor sum** 快速求;A 為全集或恰缺一值要特判。
