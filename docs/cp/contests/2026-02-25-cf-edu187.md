### [A. Towers of Boxes](https://codeforces.com/contest/2203/problem/A)
`貪心` `數學` `~800`

::: info 題意
有 $n$ 個相同箱子,每箱重 $m$、耐重 $d$。要把全部箱子疊成若干塔(每塔至少一箱),限制:每箱上方的總重不得超過 $d$。

**求**:最少塔數。

例:$m=2,d=5,n=7$ → 一塔至多 $3$ 箱(底箱上方 $2$ 箱重 $4\le 5$;疊 $4$ 箱則 $6>5$)→ $\lceil 7/3\rceil=3$ 座。
:::

::: tip 作法
一塔至多 $k=1+\lfloor d/m\rfloor$ 箱(**瓶頸只在最底箱:$(k-1)m\le d$**),答案 $\lceil n/k\rceil$,$O(1)$。
:::

### [B. Beautiful Numbers](https://codeforces.com/contest/2203/problem/B)
`貪心` `數學` `~1000`

::: info 題意
定義 $F(x)$ 為數位和;$x$ 是 beautiful ⇔ $F(F(x))=F(x)$。給定十進位整數 $x$,一步 = 把某一位換成別的數字,結果不得有前導零。

**求**:把 $x$ 變 beautiful 的最少步數(可為 $0$)。

例:$x=993$(和 $21$)→ 一步最多減 $9$,$21-9=12>9$ 不夠;兩步 $993\to 903\to 103$(和 $4$)→ 答案 $2$。
:::

::: tip 作法
**beautiful ⇔ 數位和 $\le 9$**($F$ 的不動點恰為 $0\sim 9$);每位的減量 = 該位數字(首位只能換成 $1$,減量少 $1$),減量由大到小貪心扣到和 $\le 9$ 為止。
:::

### [C. Test Generator](https://codeforces.com/contest/2203/problem/C)
`位元` `二分` `貪心` `~1500`

::: info 題意
給定 $s,m$(由參考 code 掃 $60$ 個位推得 $s,m<2^{60}$)。構造非負整數陣列:$\sum a_i=s$ 且每個 $a_i\,\&\,m=a_i$(即 $a_i$ 是 $m$ 的 submask)。

**求**:判斷是否存在;存在則輸出最小長度 $n$(無解輸出 $-1$)。

例:$s=13,m=5$ → submask 只有 $\{0,1,4,5\}$:$5+4+4=13$ 可行;$n=2$ 上限 $5+5=10<13$ → 答案 $3$。
:::

::: tip 作法
固定 $k$ 個數時,**$m$ 的每個 set bit 全體總共最多被用 $k$ 次**(每個數各位至多一次)→ 高位往低位掃,維護欠額 $f$(單位=當前位):$f\leftarrow 2f+s_i$;遇 $m$ 的 set bit 扣 $f\leftarrow f-\min(f,k)$(**高位能吃就吃**,留給低位需求會倍增;cap 在 $f$ 保證不超湊)。掃完 $f=0$ ⇔ 可行,對 $k$ 單調 → 二分最小 $k$。

```cpp
bool ok(ll s, ll m, ll k) {              // k 個數湊不湊得出 s
    ll f = 0;                            // 欠額(單位 = 當前位)
    for (int i = 59; i >= 0; i--) {
        f = 2 * f + (s >> i & 1);
        if (m >> i & 1) f -= min(f, k);  // 這一位全體最多用 k 次
    }
    return f == 0;
}
// 無解 ⇔ !ok(s, m, 2^60);否則二分最小 k
```
:::

### [D. Divisibility Game](https://codeforces.com/contest/2203/problem/D)
`博弈` `數論` `貪心` `~1700`

::: info 題意
給定陣列 $a$($n$ 個)與 $b$($m$ 個),$1\le a_i,b_i\le n+m$(值域由參考 code 推得)。輪流操作、Alice 先手:當前玩家選 $x\in a,y\in b$,Alice 須 $x\mid y$、Bob 須 $x\nmid y$,然後從 $b$ 移除一個 $y$($x$ 留在 $a$)。

無法行動者輸。**求**:雙方最優下誰贏。

例:$a=[1,2],b=[2,3]$ → $2$ 被全體整除(Alice 專屬),$3$ 兩人皆可拿($1\mid 3$、$2\nmid 3$)。Alice 先拿公共的 $3$,Bob 立刻無牌可動而敗;若 Alice 先拿自家的 $2$,Bob 拿走 $3$,反而 Alice 敗。
:::

::: tip 作法
按 $d(y)=|\{x\in a:x\mid y\}|$ 把 $b$ 三分:$d=n$ → Alice 專屬、$d=0$ → Bob 專屬、其餘公共。**雙方最優都是先燒公共堆、再動自家專屬堆**(拿公共牌同時消耗對手資源,交換論證),之後 $O(n+m)$ 模擬到有人動不了;$d$ 用調和級數篩,$O((n+m)\log(n+m))$。

```cpp
for (int i = 1; i <= n + m; i++) if (cntA[i])       // 調和級數篩 d(y)
    for (int j = i; j <= n + m; j += i) d[j] += cntA[i];
// F += (d[y]==n), S += (d[y]==0), C += 其餘
// 輪流:有 C 拿 C,否則拿自家專屬;拿不到者輸(直接模擬)
```
:::

### [E. Probabilistic Card Game](https://codeforces.com/contest/2203/problem/E)
`機率期望` `BIT` `二分` `~2200`

::: info 題意
$m$ 輪遊戲:第 $i$ 輪先加入一張值 $a_i$ 的牌(保證值兩兩相異);若牌數 $\ge 3$,Alice 選牌 $a$,Bob 看過後選另一張 $b$,再從其餘 $i-2$ 張均勻隨機抽 $c$(選完全放回牌堆)。

Bob 得分:若 $|a-c|\le|b-c|$ 或 $c$ 嚴格介於 $a,b$ 之間則 $0$,否則得 $|b-c|$。Alice 最小化、Bob 最大化期望。**輸出**:每一輪(牌數 $\ge 3$ 時)的最優期望,mod $998244353$。

例:牌堆 $\{1,2,3,10\}$ 的那輪:Alice 出 $3$,Bob 最佳拿鄰居 $2$,僅 $c=1$ 得 $|2-1|=1$ → 期望 $\frac12$;Alice 若出 $2$,Bob 拿鄰居 $3$ 後 $c=10$ 可得 $7$,差很多。
:::

::: tip 作法
**Bob 的最佳回應必是 $a$ 的相鄰牌**:拿右鄰 $q$ 得 $\sum_{c>q}(c-q)$,拿更遠只會項項變小(左側對稱)。故 Alice 出名次 $pos$ 的牌代價 $=\max(L(pos),R(pos))$,且 **$L$ 隨名次遞增、$R$ 遞減** → 二分交點,只需檢查交點與其左鄰;值域 BIT 前綴和 + order-statistics(查名次/第 $k$ 小),每輪 $O(\log^2)$,最後乘 $\text{inv}(n-2)$。

```cpp
auto L = [&](int pos, auto it) {      // Bob 拿左鄰 pred:Σ_{c<a}(pred - c)
    return it == cur.begin() ? 0LL
         : xs[*prev(it)] * 1LL * pos - pref(*it - 1);
};
auto R = [&](int pos, auto it) {      // Bob 拿右鄰 succ:Σ_{c>a}(c - succ)
    return next(it) == cur.end() ? 0LL
         : (sum - pref(*it)) - xs[*next(it)] * 1LL * (n - pos - 1);
};
// 二分最小 pos 使 L ≥ R;ans = min(max(L,R)) at {pos-1, pos}
// 每輪輸出 ans * inv(n - 2)  (mod 998244353)
```

(留言區補充:最優點其實落在「左右和最平衡」處,可 $O(n\log n)$。)
:::

### [F. Binary Search with One Swap](https://codeforces.com/contest/2203/problem/F)
`分治` `組合計數` `貢獻` `~2600`

::: info 題意
給定 $n$,考慮標準二分搜尋($m=\lfloor\frac{l+r}2\rfloor$,依 $a_m$ 與 $x$ 大小縮半邊)在 $a=[1,2,\dots,n]$ 上執行。對每對 $1\le i<j\le n$,把 $a_i,a_j$ 交換後,定義 beauty = 仍能被找到的 $x\in[1,n]$ 個數。

**輸出**:對每個 $k=0,\dots,n$,beauty 恰為 $k$ 的 pair 數 $p_k$。

例:$n=7$、swap $(2,6)$ → 逐一跑二分,找不到的恰是 $\{2,3,5,6\}$,beauty $=3$(結構見作法圖)。
:::

::: tip 作法
二分搜尋的遞迴是一棵樹;**同長度的區段完全同構,且出現過的長度只有 $O(\log n)$ 種**(每層至多兩種)。每對 $(i,j)$ 在樹上有唯一「分岔節點」$m$(第一個把兩位置分開的 mid),按分岔節點分類,每種長度只算一次損失直方圖、乘上出現次數 $cnt[len]$。

損失(找不到的 $x$ 數)有封閉形式:swap $m$ 與左子樹位置 $i$ → 損 $m-i$;與右子樹第 $t$ 個位置 → 損 $t$;$i,j$ 分居左右子樹 → 損 $=($**$i$ 子樹內 $i$ 右側元素數 $+1)+(j$ 子樹內 $j$ 左側元素數 $+1$**$)$,左右各只有 $O(\log)$ 種損失值,小卷積合併。不同長度總和 $O(n)$ → 整體近線性(留言區另有嚴格 $O(n\log n)$ / $O(n)$ 解)。

```cpp
calc(0, n - 1);                        // 二分遞迴:cnt[len]++,len 僅 O(log n) 種
for (int len : lens) {
    auto g = get(len);                 // g[t] = 此長度內損失恰為 t 的 pair 數
    for (int t = 0; t <= len; t++)
        ans[n - t] += cnt[len] * g[t]; // beauty = n - 損失
}
// get:m×左子樹 i → 損 m-i;m×右子樹第 t 個 → 損 t;
//     左×右 → 損 (i 子樹右側數+1)+(j 子樹左側數+1),兩分佈小卷積
```
